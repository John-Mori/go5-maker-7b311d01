## 3. 改善設計

### 3.1 判定ルーブリックv1 (selection-rules.mdへ)

**E直行ゲート**(1つでも該当なら見送り。スコアリング前に判定=評価コスト削減):
- G1: 電車内基準(「電車内で見られて困らないコマだけ使う」)を満たすコマが3枚未満。1コマ目・サムネは必ず基準内(CLAUDE.md §6.1)
- G2: 同一作品が対象chで使用済み。acc1/acc2間の同一作品は原則回避(コマ・文言を必ず変える条件つきでのみ可)
- G3: 5秒で意味が伝わる素材が無い(persona_detail §5-E)
- G4: 無料流通が広範で購入動機が崩れている

**素材スコア(星南・5点)**: 停止力(1コマ目が0.5秒で目を止めるか)0-2 / 未解決の疑問を作れるか0-1 / 基準クリア素材量(4-5枚=0.5・6枚以上=1)0-1 / 差分・違和感の見えやすさ0-1

**採算スコア(クラウディア・5点)**: 割引率(30%以上=1・50%以上=2)0-2 / 販売数sales_n 0-1 / レビュー(件数×平均)0-1 / 今出す理由(セール期限・新作鮮度)0-1

**判定表**:
| 判定 | 条件 |
|---|---|
| A(即制作) | ゲート全通過+素材4以上+採算3以上 |
| B(限定テスト) | 素材4以上+採算1-2(集客テスト) / 素材3+採算4以上 / **二人の意見が割れた時**(persona_detail §5-B「どちらかが勝つのではなく小規模テストへ」) |
| C(セール待ち) | 弱点が価格・時期だけで、セール到来で解ける |
| D(素材保存) | 素材2-3で構図・訴求の参考価値あり |
| E(見送り) | ゲート該当 or 素材0-1 |

- **成約数は構造的に観測不可**(agents定義の鉄の掟)。採算はクリック誘発力・市場の強さ(販売数・レビュー)で語る。「用途: 集客型/成約型」の分類は維持するが、成約数を根拠にしない。
- ★**全閾値は暫定**(実データゼロの状態で設定)。§3.5の突合で判定10件ごとに較正し、版数を上げる。
