# 価格仮説 (product-scout)
> 運用: 結論先頭・1項目=数行・新しい項目を上へ。この部門だけが更新する(他部門はinsight経由で提案)。

## 突合仮説の様式

判定時の素材スコア・採算スコアと、投稿後のshorts-analyst数字(再生・クリック)のズレを記録し、selection-rules.md閾値の較正根拠にする。

**様式**: 観測(実数)→ズレの仮説→ルーブリック改善案。フィールド=日付/cid/判定(A-E)/判定時スコア/実績数字/ズレの内容/仮説/反映案。例形式は以下のとおり。

**例(架空)**: 2026-07-20 / cid:12345678 / A判定(素材5/採算4) / 再生数260(予想800)・クリック率1.2%(予想3%) / 素材スコアが高いのに数字が弱い / 「停止力が高いが未解決の疑問が弱かった。この作者のスタイルは1コマ目で引いて中盤で転換するタイプで、5秒フォーマットに向かない」 / selection-rules v1の「未解決の疑問」項(0-1)の定義を精密化。具体例を追加。

較正トリガーはズレ件数3件またはA-E判定累計10件のいずれか早い方で発火。根拠つきでselection-rules.mdの版数を上げ、変更箇所と理由を同ファイルの版数節に併記する。
