## E直行ゲート (スコアリング前に判定・1つでも該当なら見送り)
スコアリングより先に判定して評価コストを削る。1つでも該当したら素材・採算スコアを付けずにEとする。
- G1: 電車内基準(電車内で見られて困らないコマだけ使う)を満たすコマが3枚未満。1コマ目・サムネは必ず基準内でなければならない(CLAUDE.md §6.1)。
- G2: 同一作品が対象chで使用済み。acc1とacc2間の同一作品は原則回避する。コマ・文言を必ず変える条件つきでのみ可。
- G3: 5秒で意味が伝わる素材が無い(persona_detail §5-E)。
- G4: 無料流通が広範で購入動機が崩れている。

## 素材スコア (星南・5点満点)
1つでもゲート該当なら採点しない。各項目の最大点を超えない。
- 停止力: 1コマ目が0.5秒で目を止めるか。0-2点。
- 未解決の疑問を作れるか。0-1点。
- 基準クリア素材量: 4-5枚=0.5点・6枚以上=1点。0-1点。
- 差分・違和感の見えやすさ。0-1点。

## 採算スコア (クラウディア・5点満点)
割引率は非累積(50%以上は2点で確定・1点と足さない)。
- 割引率: 30%以上=1点・50%以上=2点。0-2点。
- 販売数sales_n。0-1点。
- レビュー: 件数×平均で評価。0-1点。
- 今出す理由: セール期限・新作鮮度。0-1点。

## 判定表
| 判定 | 条件 |
|---|---|
| A(即制作) | ゲート全通過+素材4以上+採算3以上 |
| B(限定テスト) | 素材4以上+採算1-2(集客テスト) / 素材3+採算4以上 / 二人の意見が割れた時(小規模テストへ・persona_detail §5-B) |
| C(セール待ち) | 弱点が価格・時期だけで、セール到来で解ける |
| D(素材保存) | 素材2-3で構図・訴求の参考価値あり |
| E(見送り) | ゲート該当 or 素材0-1 |

## 注記
- 成約数は構造的に観測不可。採算はクリック誘発力・市場の強さ(販売数・レビュー)で語る。
- 「用途: 集客型/成約型」の分類は維持する。ただし成約数を根拠にしない。

## 引き継ぎ経路 (裁定3A)
- A/B判定は部門chでPacket報告する。
- Chamiが最終採用を決める(persona_detail §7の原則)。
- 採用後、研究室がD1へ記録(candidate.recommended)し、copy-directorへdispatchする。D1書き込みとdispatchは研究室の権限であり、この部門は行わない。
- 採用されたPacketはqa-reviewerがURL・価格・期限・重複・欠落を確認する(persona_detail §8)。報告に「QA確認依頼」の1行を含める。

## 定期スキャン (裁定4・2026-07-18 Chami承認「ちゃんと毎朝欲しい」)
- 頻度: **毎日・朝に1回**。朝を逃した日(閉窓等)は、その日最初の起床時に**必ず遅延実行**する(遅れても抜かさない)。
- 母集団の定義(2026-07-18実測): D1 go5_fanza worksの実体=①Chamiの追跡サークル(tracked_makers・現在2件=千暮ゆき/アビィロード)の作品の実売追跡(PC側が3時間ごと自動取得) ②Chamiが候補タブへ追加した作品の詳細(価格・サンプル画像)。**市場全体の新作は含まれない**。母集団を広げる手段=追跡サークルの追加(アプリ操作のみ)or市場巡回の新設(改修案件)。
- 恒久化: 窓の開閉に依存しない毎朝実行は常駐への組み込みが必要=system-engineer要件としてrouter経由で依頼済み(2026-07-18)。実現までは上記の遅延実行で担保する。
- 手順: works表から発売90日以内をsales_n降順で読む→使用済み台帳と照合(E直行ゲートG2)→採算スコアで予選→通過した上位最大3件を「採算予選通過・素材確認待ち」として部門chへ提案(クラウディア名義)
- 予選通過ゼロの日は**提案を出さない**(ノイズゼロ)。
- 制約: セール期限・レビューはD1に無い=提案時に「追加確認事項」として明記。素材スコアはサンプル画像(info_json内のsampleImageURL)の確認後に確定し、A-E最終判定は素材+採算が揃ってから。
- この窓が開いている日に限る(閉窓日は飛ぶ)。

## 報告様式
