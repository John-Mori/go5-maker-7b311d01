# FAQ知識パック (無料応答陣向け・育てながら使う)

## 目的
ローカルqwen/Gemini受付など無料応答陣が「Chamiのよくある質問」に一次回答できる正答率を上げ、
高価なClaude司令塔へのエスカレーションを減らす(=Claudeトークン節約)ための知識源。
`インシデント.md`と同じく、確証のある事実だけを承認しながら育てる。

## 育て方のルール
1. **出典必須**: 各回答は既存ドキュメントに明記されている事実のみから作る。各エントリに出典(ファイル名+節)を必ず付ける。
2. **承認制**: Chami/司令塔の承認を経て追記する(インシデント.mdと同じ運用)。無断で憶測を足さない。
3. **曖昧なら入れない**: 出典を辿っても断定できない項目・未確認事項は採用しない。

## フォーマット
`### Q. <質問>` の後に1〜4文の簡潔な回答、末尾に `(出典: <ファイル §節>)` を付ける。

---

### Q. コードを直したのに古いまま表示される。どうすればいい?
`index.html`のアセット参照は`app.js?v=N`の形。中身を変えたら`N`を1つ上げるとスマホで確実に最新が読まれる。公開は`git add -A && git commit && git push`後、GitHub Pagesへ1〜2分で自動反映される(フロントのpushは司令塔に委ねる)。
(出典: CLAUDE.md §3「キャッシュ運用」/§4「ビルド・テスト・公開」)

### Q. GAS(コード.gs)を直したらどう反映する?
`gas/コード.gs`を編集したら必ず`GAS_VERSION`をバンプし(日付+英字サフィックス)、`GASを反映.bat`(=`node scripts/deploy_gas.mjs`)を実行するだけ。claspが既存デプロイIDに新バージョンを当てるためexec URLは不変で、GASエディタへの手動コピペ・手動再デプロイは不要。
(出典: CLAUDE.md §4「GAS(コード.gs)の反映」)

### Q. FANZAの直リンクをYouTubeの説明欄や固定コメントに貼ってもいい?
だめ。正しい導線は「YouTube → 短縮URL(r2/da.gd) → Bluesky投稿 → FANZA」の2段クッション構造で、生アフィリンクが出てよいのはBluesky投稿本文のみ。チャンネル喪失を防ぐための規約防衛ルール。
(出典: CLAUDE.md §6.1「規約防衛」)

### Q. 同じ動画を月詠み(acc1)と宵桜艶帖(acc2)の両方にそのまま投稿してもいい?
だめ。2チャンネルへ同一動画をコピー投稿するとinauthentic/スパム判定(2025/7収益化ポリシー)の火種になるため、作品・コマ・文言を必ず変える。
(出典: CLAUDE.md §6.1「規約防衛」)

### Q. なぜ生成される動画には音がついていないの?BGMは焼き込めない?
仕様。音源は動画ファイルに焼き込まない設計で、BGMはYouTubeアプリのShortsエディタで「許可された音楽」を付けるのが正式運用(トレンド音源を合法に使える唯一の経路)。将来もBGM焼き込み機能は作らない方針(改善書S-2で不採用)。
(出典: CLAUDE.md §6.1「規約防衛」)

### Q. Bluesky投稿の「クリック数」は何を数えている?
投稿の短縮URL(Bitly)の開封数。アフィリンクは本文に生のまま(無改変)貼るためaf_id計測は壊れないが、生アフィリンクのクリック実数自体は取得できない(FANZA管理画面の数字が正)。
(出典: CLAUDE.md §3「投稿記録&クリック集計(§10機能)の要点」)

### Q. UI文言の括弧は全角と半角どちらを使う?
半角`()`のみ。全角`（）`は禁止。
(出典: CLAUDE.md §1「現在の状態(最新)」冒頭)

### Q. 「直したのに反映されない」と言われたら、まず何を疑う?
まずコードのバグより配信層を疑う。GitHub Pagesは旧ファイルを最大10分キャッシュ配信することがあり(対策済みだが再発しうる)、さらにpush成功はビルド成功を意味しない(過去にJekyllのYAML解析エラーで特定コミットだけ配信に失敗した実例あり)。liveの版数を実測して切り分ける。
(出典: インシデント.md INC-28「『直したのに反映されない』の正体は配信キャッシュ」/ INC-66「実装したのに反映されない」はまず配信層を疑う)
