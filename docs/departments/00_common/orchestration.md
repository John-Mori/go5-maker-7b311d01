**ルール(トークン浪費の断ち方):**
- **キャラ口調は「担当サブが最初から口調込みで書く」。研究室(Opus/Fable)を口調変換に使わない**(別途の口調変換パスを作らない=Chamiが指摘した「無駄なトークン」の発生源を断つ)。部門報告は担当モデル(既定haiku)がそのキャラの声でそのまま出力する。

## ★質問・選択肢・承認要求も必ずDiscordへ出す (2026-07-16 Chami指示・上と対になる規約)

   - **★大原則はその手前にある: そもそも貼らせない**。人間に長文を正確に貼らせる運用は必ず壊れる(文字化け・貼り先違い・外出中は不可)。**機械が実行できるものは機械にやらせる**——例=部門の窓は `scripts/_daemons/open_dept_window.ps1` が起動文を組み立てて `claude` へ直接渡す(Chamiは「改修αの窓を開けて」と言うだけ)。承認の常用パターンは `.claude/settings.local.json` のpermission ruleで事前に通す。
   - **根拠の4つ目(QA発見・2026-07-17): 貼らせるものは古くなる**。起動文の3段印対応を入れた数時間後には、Chamiの手元にある起動文が全部2世代旧版になっていた(QAが機械で数えて確認: `local/_work/`の名指し0箇所・既読0箇所)。人間の手元に置いた文面は更新が追いかけない。機械が毎回組み立てれば(`dept_boot_prompt.py`)常に最新が出る。
   - 貼らせるのは**それでも人間の手が要る一点だけ**に限る(chatペインでの承認など)。
