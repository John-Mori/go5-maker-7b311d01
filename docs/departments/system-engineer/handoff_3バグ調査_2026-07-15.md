## ② Books の%オフ表示が誤り 【根本原因=高確度・確定】
- 原因: FANZA **Booksは価格データ(DMM API `prices` / 商品ページ JSON-LD `offers.price`)に時限割引が乗らない**(値引きはカート適用)。同人ページにはこの癖が無い。→ `discountPct=round((1-price/listPrice)*100)` が0/誤りになる。
- 対策の実装は **PC側 `scripts/fetch_missing_works.mjs` の `scrapeBookPage()`(L97-107)にだけ存在**: ページの `>◯%OFF<` バッジから割引後価格を逆算(`price=round(listPrice*(100-pct)/100)`)。
- **ライブを叩く2経路に欠落**: `fanza-worker/src/index.js` の `fetchViaApi()`(L427-495, ebookもdoujin扱いでAPI値を素通し) と `scrapeFanzaItem()`(L533-647, 同人用regex流用・`service_name/floor_name`を"同人"にハードコード)。
- 修正案: fetch_missing_works.mjs のバッジ逆算を fanza-worker の scrapeFanzaItem(Books URL時)へ移植 + fetchViaApi の ebook 後処理として追加(要: on-sale Books cidで生`prices`を実確認) + `service_name`のハードコード"同人"を実ソース反映。
- **未確認**: 公式API経路(fetchViaApi)が同じ癖を持つか(要 live DMM API 照会)。
- **保留理由**: Cloudflare Worker の改修+反映(wrangler)=データ基盤の変更。Chamiの「Worker直していい?」GO待ち。

## ① 投稿編集→動画生成へ で候補のコメント/画像が消える 【根本原因=未特定】
- `#refImgToMovie`(`candidates.js:931-942`)は遷移前に `refImgSave(it.cid, pending)` で**明示的に保存**しており、静的レビューでは消す処理が見当たらない(該当パスは正しく見える)。
- **確定した別バグ**: `applyWorkUrl_`(`candidates.js:1123-1138`・"保存"ボタン=作品URL編集時)が refimg再保存で **`memo` と `twitterUrl2` を落とす**(union漏れ)。→ これは低リスクで直せる(保存オブジェクトに memo/twitterUrl2 を追加)。ただし報告の「動画生成へ」とは別ボタン。
- 仮説(未確認): (1) `refImgSave`のIDB書込がfire-and-forgetで静かに失敗(quota/privateモード)。(2) **端末間同期の競合**: `core/sync.js` の ref/bsky/post は whole-object last-write-wins(`mergeMaps` L166-171)で、cand_items にはある per-field union+tombstone 保護が**無い**(INC 2026-07-15)。別端末の古い同期でレコード全体(コメント+画像)が上書きされ得る。
- **Chami確認待ち**: 単一端末でも起きる?複数端末同期時?押下直後に消える?再読込後に空? → (1)か(2)か切り分け。
- 修正案(確認後): 単一端末なら refImgSave を await 化/失敗を可視化。同期競合なら ref/bsky/post にも cand_items 相当の per-field union+tombstone を移植(データ層・要注意)。
