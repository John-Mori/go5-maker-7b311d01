## 1. 絶対規約(破ると"直された"になる・最優先)

- **「よくあるClaudeの紫デザイン」を使わない(恒久・Chami指示2026-07-03)。** モーダル/オーバーレイ/
  ボタン等の新規UIに紫を使わない。禁止色の例: `#5b3f8e / #2a1a4e / #1a1a2e / #d4b3ff / #c4a0ff`。
  → アプリ配色(§2)に合わせる。※アカウントテーマの紫(月詠み=#241a3d 等)は**別枠でOK**。
- **UI文言の括弧は必ず半角 `()`。全角 `()` 禁止**(CLAUDE.md・system-engineer BOOT)。
- **時刻表示は日本時間(JST)**。`toISOString()`/UTCをそのまま表示に出さない(Chamiの体感と9hズレる)。
  JSは端末ローカル(`getHours()`等)。例外=Cloudflare枠リセット等のUTC固定値は集計UTC・表示JST併記。
- **座標系を崩さない(§3)。** px/vh/vw を直接使わない。プレビューと書き出しは同一Canvas・同一式。
- **締めの言葉に「刻んだ」系を使わない**(2026-07-16・全キャラ廃止)。記録は「(自然な完了の言葉)。(ファイル名)」。

## 2. 配色(アプリの世界観)

| 用途 | 値 | 備考 |
|---|---|---|
| ダーク面(カード/パネル) | `--card` / `#0e1422` | 背景の基調 |
| アクセント | `--accent` / `#2bb3c0`(ティール) | ボタン強調・アクティブ |
| クリーム面 | `#fffdf6` | 明るい面・コントラスト |
| アカウントテーマ紫(月詠み) | `#241a3d` 等 | ★アカウント別テーマは別枠でOK(禁止の"Claude紫"とは区別) |

- 基調はダークUI。ゴテつかせない・派手な原色を避ける。CSS変数(`--card`/`--accent`等)があればそれを使う。

## 3. 座標系(動画Canvasの最重要規約・崩さない)

- 基準フレーム **W=1080 / H=1920(9:16)が唯一の基準座標系**。`<canvas width height>` も一致。
- 位置・フォントサイズは `H×係数` / `W×係数`。旧絶対px定数は `U(v)=v*H/1280` で換算。
- **プレビューも書き出しも同一Canvasに同じ式で描画** → PC/スマホ/書き出しで一致(CSSは9:16を一様縮小表示するだけ)。
- 縦オフセットは `OFF` オブジェクトで一元管理(`OFF.whole`/`OFF.textAuthor|Detail|Title`等)。
- `document.fonts.ready` 後に再描画(フォールバックフォント計測ズレ防止)。

## 4. コントロール/操作系の好み

- **位置調整は ＋/− ボタン式(スライダーではない)**。プレビュー横で見ながら微調整(スマホは sticky)。
  (v11でスライダー→＋/−へ刷新。Chamiの確定好み)
- 段別に独立制御(①作者名/②誘導文/③大タイトル)。帯は各段のテキストに統合(帯だけ別に動かさない)。
- **2行モード**: 作者・大タイトルに「2行モード」チェックボックス。ユーザーの改行位置で2行分割・中央揃え。
- localStorage で保存/復元、「既定値に保存」「リセット」を用意する(汎用 `Store` ヘルパ)。

## 4.5 配置・幅(★Chami指示2026-07-20「ページやボタンの配置、幅、などに専念」=本部門の主戦場)

### 確立済み(迷わずこれに寄せる)
- 位置調整は **＋/− ボタン式**(スライダー禁止)。プレビューを見ながら調整できる配置に置く。
- プレビューは**スマホでは sticky** で追従させる(調整中に見失わせない)。
- 段別に独立制御(①作者名/②誘導文/③大タイトル)。帯は各段のテキストに統合。
- 主要操作は**縦一列で親指の届く範囲**に(iPhone片手操作が前提)。
- 保存/復元・「既定値に保存」・「リセット」を必ず用意する。

## 5. モバイル/レイアウト

- **iPhone/iPad のブラウザ最優先。縦型 9:16。** スマホで横潰れしない(CSSの9:16固定を崩さない)。
- タブ構成の実績: 🎬動画作成 / 📅カレンダー / 🦋投稿 / 🔗アフィリンク(iframe埋め込みでCSS/JS衝突回避)。
- キャッシュ運用: アセット参照は `app.js?v=N`。中身を変えたら **`node scripts/bump.mjs` で一括バンプ**(手動sed禁止)。

## 6. トーン/文言

- 煽り文言を書かない(規約防衛・P0)。「続きは…」型の誘導は禁止。
- 説明文・ボタンラベルは簡潔・日本語・半角括弧。過剰な絵文字装飾はしない(タブの識別絵文字は可)。

## 99. 育成手順(このKBを"成長"させる)

1. Chamiがデザイン/UI/配置/幅/色/ボタンの指示を出したら、フロント部門は作業後にここへ1行追記する。
   - 形式: `- (日付) <対象>: <確定した好み or 直された内容>`(良い例/悪い例を分けて書く)。
2. 「一発で合った」ものは§1-6の該当箇所を強化。「直された」ものは"直後に悪い例"として明記し二度踏まない。
3. 迷う境界は勝手に凝らず、ここに寄せる。無い項目はChamiに1問だけ確認し、答えを§へ恒久化する。
