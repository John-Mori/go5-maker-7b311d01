# コピー原則 (copy-director)
> 運用: 結論先頭・1項目=数行・新しい項目を上へ。この部門だけが更新する(他部門はinsight経由で提案)。

- **④コメントの口調はチャンネルで変えない(Chami裁定2026-07-18・msg 1527861174677274775)**: chテーマに載せたコメントは**データのノイズになる**。コピー評価の主眼=どのチャンネルでも**再現性のある**クリック数(アクセス数)と成約の向上(最低限の体裁は整える)。旧・ch別ボイス分け(月詠み=口語ツッコミ断言調/宵桜=しっとり体言止め)は**廃止**→rejected-patterns.mdへ記録済。C-6でraw/戦略書から要点抽出する際もこのルールは持ち込まない(上書き済扱い)。
- **チャンネルテーマは「ガワの世界観」だけ(Chami裁定2026-07-18・msg 1527856790270640298)**: 月詠み/宵桜のテーマ分けは導入やネーミングの美学=Chamiの趣味であり、**コメントの決め方・作品選定はどちらのチャンネルでも関係ない**(今後3ch目ができても同様)。※P0-3の同一文言使い回し禁止(規約防衛)は別枠で継続。
- **オリジナル命名の正本=Chamiの命名ガイド**(local/copy-director/命名ガイド_汎用_Chami正本.md・2026-07-18受領)。世界観名・サブコンテンツ名・キャラ名等の命名タスクでは必ず参照: 5原則(魂・響き・可読・唯一・没を転用)+言語選択の上位ルール(対象顧客から逆算)+響きの設計法(母音の流れ/モーラ/子音の手触り/連結/開音節)+触発からの昇華法(核を抽出→ずらす→昇華、崩れるなら敬意あるオマージュ)。研究室へもエスカレーション済(Chami指示)。

## 執筆ガードレール: 物理制約(2026-07-18シード・設計書C-2)

基準フレーム1080×1920・テキスト最大幅972px(W×0.9)。字数は全角1em換算の**暫定近似**(measureText実測で確定次第更新)。

| 対象 | 執筆上の上限目安 | 挙動 | 出典 |
|---|---|---|---|
| ④コメント(大タイトル)1行 | **10字までは縮小なし**。超えるほど視認性低下、約46字で下限21px→はみ出し | 972px超で黙って自動縮小・警告なし | app.js:276-283,332,358 |
| ④コメント2行モード | 各行10字前後を狙う | 最初の改行で分割・広い方に合わせ縮小(下限21px) | app.js:286-296,354 |
| ②誘導文 | **約18字が絶対上限**(約20.8字で画面外) | 折返しも縮小も無い。長文は帯ごとキャンバス外へ | app.js:241-273,347 |
| ①作者名1行 | 1行約20字。prefix(作者：/引用：)は自動付与=入力に書かない | 文字単位折返し(縮小なし)・行ごとに帯 | app.js:152-165,337-338 |
| ①作者名2行モード | 各行短め | 両行を1枚帯・縮小下限16.5px | app.js:341-342 |

| Bluesky本文 | **自由記述の実効目安≈180〜210字**(自動付与分90〜120字を差引) | 合成順=本文→PR行→アフィリンク(短縮は取得成功時のみ)→セール行(常時ON)。300グラフェムのハードチェックは「今すぐ投稿」経路のみ=自衛必須 | bluesky.js:652-667,1660-1662 |

- コメント型①〜⑧と目的(成約/集客)は生成前の必須選択。型はメタデータで描画に影響しない=型の選定理由はコピー案に文章で添える。
- コメントの字数目安は10〜18字(暫定・検証対象)。

## 執筆ガードレール: 規約防衛(P0・チャンネル喪失防止)

- 煽り文言禁止(「YouTubeでは見せられない続きは…」型=名指しの違反パターン)。禁句: 「概要欄」「続きは」等の直接誘導語・露骨な性的単語。
- 電車内基準: 提案するフック画像・1コマ目は「電車内で見られて困らないコマだけ」。
- 2ch差別化: acc1(月詠み)/acc2(宵桜艶帖)へ同一文言を出さない(同型量産はinauthentic判定の火種)。※差別化は「同じ文言の使い回し禁止」であって「ch別の口調作り分け」ではない(口調のch分けは廃止=Chami裁定2026-07-18・冒頭項参照)。
- FANZA直リンクはYouTube側に絶対に貼らない(2段クッション構造)。
- 投稿系の相談にはgo5-post-guardスキルを先回りで通す。境界事例は自己判断で通さずqa-reviewer/研究室へ。

```
  狙い: (1行)
推奨: (どれか+理由。三笘の評価=視覚/字数、芽衣の評価=感情の当たりどころ)
検証: (採用後に見る指標1つ。集客=engagedViews系/成約=短縮URLクリック。成約数そのものは観測不可なので約束しない)
ガードレール: 検査済み(物理/規約)。境界事例は上げた旨を明記
```

## 知見記録の型(1タスク1行以上)

- copy-rules.md: 「(ルール)。根拠=(INC/成績/Chami指示+日付)」
- winning-patterns.md: 「(目的×型×カテゴリ)で『(コピー)』→(指標と数字)。(日付)」
- rejected-patterns.md: 「『(コピー)』は(理由)で不採用/低成績。(日付)」
- Chamiに修正されたらその日のうちに修正前→後と傾向をここへ。

## 過去INCからの執筆教訓

- 題名は事実上の識別子(INC-72: 題名一致でデータ復元)=投稿後の題名変更は照合系に副作用。変更提案時は必ず注記。
- コピーの自動挿入機能を提案する時は「対象ごとに1回だけ」の冪等ガードを要件に含める(INC-50)。
- 前回値の残留は取り違え事故のもと(INC-35/70)=「前回と同一でないか」の確認を挟む。
