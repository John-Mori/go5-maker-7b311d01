## 教育の中身 (現行の手段と方針)
1. **知識パックが主教材**: 正本=`docs/departments/00_common/system-brief.md` → `scripts/llm/build_knowledge.py` で `local/llm/knowledge.md` を生成。**規約・仕様が変わったらパックを作り直すのが「教育」のセット**(2026-07-16に怠りが発覚→是正済)。ローカルの賢さはモデルではなくパックで決まる(実証: パック無しでは「自動投稿機能がない」と嘘をついた/パック有りでは正答+不明分は研究室へ回す判断まで出来た)。
2. **採点と学習還元** (Chami指示2026-07-15): ローカルの実応答の質・ミスを採点し、弱点をパック改訂(またはFAQ追記)へ還元する。横断事項は研究室経由。
3. **将来**: LoRA等の微調整(llm-qa部屋の領域と連携)。深い技術検証はllm-qa、**教育の実施はここ**。

## 近隣部屋との区分け
- **llm-qa(質問-chamiのローカルllm学習ルーム)**: Chamiが技術を学ぶ部屋(Ollama/qwen/ComfyUI/SD等の質問)。
- **llm-growth(ローカルllm成長進捗)**: 成長記録・評価の**見える化**の部屋。教育の実施はllm-edu。
- **ホイミン(gemini)**: 別の受付AI。教育対象はあくまでローカルqwen(ただしパックは共用なのでホイミンにも効く)。
