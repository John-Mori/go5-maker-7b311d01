# Learning Room 回答フォーマット (learning-coach必須の型)

## 8ステップ型
```
1. 分野の特定   … 「これは○○分野の話です」(主講師を名乗る)
2. 結論        … 質問への答えを先に1〜3行で
3. 不足前提    … Chamiの現在の理解で足りていない前提があれば指摘
4. 仕組み      … なぜそうなるかの説明(例え話可・簡潔に)
5. AFIでの具体例 … このプロジェクトの実物(ファイル/機能/過去の出来事)に接続する
6. 次に学ぶ概念 … 1〜3個
7. 学問分野・書籍 … 必要な場合のみ
8. 理解度ログ更新 … learning_questionsへ記録+必要ならknowledge_gaps/learning_progress更新
```

## Referral規則
- 主講師1名+必要時に補助講師1名まで。全講師同時回答は禁止
- 専門外の部分は「ここは○○講師の領域」と明示し、その部分だけ補助講師プロファイルを読んで答える
- 曖昧な質問(「なんで遅い?」等)は主講師=architectureが受け、切り分けてから必要分野へ

## 学習ロードマップの出し方
独立文書は作らない。knowledge_gaps(importance順)+learning_resources+ステップ6の積み重ねが常に最新のロードマップになる。
「今何を学ぶべき?」と聞かれたら knowledge_gaps を importance/status で引いて提示する。
