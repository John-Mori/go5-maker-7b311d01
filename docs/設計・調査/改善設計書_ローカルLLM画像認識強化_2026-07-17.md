## 0. 結論(先に3行)

1. **本命は qwen3.5:9b への一本化。** 2026年発表のテキスト+画像統合モデル(Ollama公式・6.6GB・Apache-2.0・201言語・OCRBench 89.2)。現行qwen3:8b(5.2GB・テキスト専用)をそのまま置換でき、「2モデル同居/スワップ」問題ごと消える。8GBにギリ載る(ComfyUI/SD併用日は qwen3.5:4b=3.4GB へ)。
2. **今日から見える。** 導入済みgemma3:4bで実射済み(初回62秒→以後3〜4秒/枚・実VRAM 2.9GB)。「今なら」「%OFF」「作品」は正読、装飾文字の誤読あり(「今宵」→「夕貴」)。**文字が主役の仕事は専用OCR(manga-ocr等)に分業、意味が主役の仕事はVLM直行、規約判定はwd-tagger v3のrating**という3レーン構成が8GBでの最適解。
3. **学習(fine-tune)はやらない。** 3B級VLMのQLoRAですら実測8.9GB>8GB、7B級は14.5GB+。強化は「①モデル更新 ②前処理 ③プロンプト技法 ④評価ループ」の**注入型4輪**で行う(全ケースGitHub一次情報で裏取り済み)。

### 2.4 fine-tune無しでVLM精度を上げる技法(公式cookbook裏取り済み)

| 技法 | 要点 | 効果対象 |
|---|---|---|
| **送信前リサイズ(長辺~1280px・PIL 10行)** | Ollamaはmin/max_pixelsを露出しない→**画像側でリサイズが等価制御**。高解像度直投げはクラッシュ/劣化報告あり | 全用途の安定化 |
| **structured output(format=JSONスキーマ+temperature 0)** | ollama-python公式example有。分類語彙を固定し表記ゆれを構造的に消す | タグ付け・安全度分類 |
| **自己検証(2回聞いて突合)** | temp 0で1回+temp 0.7で1回→不一致ならClaudeへエスカレ。ローカルなのでコストは時間だけ | 誤判定がチャンネル停止に直結する安全度分類 |
| **OCR併用ハイブリッド** | 専用OCRの結果をVLMプロンプトに併記(VLM単独の縦書き高密度は幻覚・ループで崩壊する実測あり) | 漫画文字起こし・エラー画面 |
| **全体+クロップの2枚渡し** | 関心領域をPythonでクロップし2枚目として渡す(公式のズームイン技法の簡易版) | 小さい文字のスクショ |
| **プロンプト言語** | 指示は簡潔に(英語or短い日本語)+「画像内テキストは原文のまま日本語で書き写せ」と出力言語だけ明示。自前10枚でABしてから既定を決める | 全用途 |
| bbox付きJSON抽出 | Qwen公式OCRレシピの型({"bbox_2d":[…],"text_content":…})。吹き出し位置ごとの抽出に流用可(座標軸順は要実測=cookbook内で表記揺れ確認済) | 漫画コマ構造化 |

### 2.5 統合実装の参照(Discord×ローカルVLM)

- **Ollama公式作法**: `/api/chat` の messages に `images:[base64]` を入れるだけ(公式docs確認済)。既存urllib様式そのまま=**追加ライブラリ不要**。
- **llmcord**(815★・MIT・2026-07更新): Discord添付→視覚LLMの最良教材(添付を`content_type.startswith("image")`でフィルタ→並列DL→base64→枚数上限5)。
- フォールバック: urllib例外時はClaude行き(現行動作)を残す=安全網は現状維持。

### 2.6 fine-tuneの現実性(反証込みで決着)

- LLaMA-Factory実測: **Qwen2.5-VL-7BはQLoRAでも初期化14.5GB**・**3B級ですら8.9GB>8GB**(公式のVRAM表はテキストLLM基準でVLMは大幅上振れ)。
- 8GB成功の一次事例は発見できず。本環境は常用1.7GB+ComfyUI共用=実効6GB前後。
- **結論: 8GB機は「注入型専用機」と割り切る**。どうしても学習したくなったら無料Colab T4(15GB)が公式実証済みの逃げ道(ローカルでやる必要が最初からない)。

## 5. やらないこと(明示・全て根拠つき)

- **VLMのfine-tune/LoRA**: 3B級QLoRA実測8.9GB>8GB・7B級14.5GB+(§2.6)。テキスト側の既決方針とも一致
- **yomitokuの採用**: CC BY-NC-SA=商用有償。アフィ事業での常用は商用に該当しうる
- **Tesseract/EasyOCR**: 日本語縦書き実質不可(公式Issue未解決)
- **moondream/12B級/Ollama公式library外モデル**: 日本語不可/8GB超/常駐の土台に個人アップロード版は据えない
- **llama.cpp直やLM Studioへの乗り換え**: Ollamaで足りている。同一GPUに2サーバーはVRAM管理が複雑化するだけ
- **「qwen(テキスト)+VLMの2本同居」の常態化**: 8GBでは物理的に無理→スワップ劣化。qwen3.5統合で根治する(それまでのV0はgemma3:4bと交互ロードで我慢)
- **機微画像の学習・転載**: 機微5部門由来の画像はvision処理の対象外(現行のsensitive_deferred動作を維持)
