## 1. コンセプト
ピクセル風の「部屋」に部門ごとのAIメンバーが座り、頭上に**本物の仕事**を吹き出しで表示するダッシュボード。
- 大原則(v4継承): **偽Status禁止**。吹き出し・状態は全てD1の実データ(dept_tasks/dept_events/system_changes)から導出
- 待機中の「自主トレ中」「野良猫と会話中」等はChami設計の**装飾演出**として表示(ツールチップで「演出」と明示=実業務と混同させない)

## 2. 画面構成
```
┌ CEO室(Chami) ─ 司令塔デスク(アメス/アロンソ常駐表示) ┐
├ 改修部 ├ 商品選定部 ├ コピー部 ├ 分析部 ┤
├ 品質管理部 ├ 学習室 ├ 報告・通知部 ├ (改善部=キャラ待ち) ┤
└ タイムライン(直近のdept_events 20件) ┘
```
- 各部屋: 部門色 / キャラ名チップ(S7までは色ブロック+名前・立ち絵ができたら差し替え) / 吹き出し=「担当タスク名+状態」
- 部屋クリック: その部門の open/in_progress タスク一覧・直近完了(result要約)・最終活動時刻
- ヘッダ: 全体サマリ(open件数・本日完了数・直近Incident・最終更新時刻)

## 3. データ導出規則(正直ステータス)
| 表示 | 出所 |
|---|---|
| 作業中バブル | dept_tasks(status=in_progress)のsummary |
| 仕事箱バッジ | dept_tasks(status=open)の件数 |
| 直近の完了 | dept_tasks(done)のresult先頭行 / dept_events直近 |
| 最終活動 | dept_events/system_changesの当該部門最新時刻 |
| 待機演出 | タスク0件時のみ。persona_manifestのidle定義からランダム(「演出」注記付き) |

## 4. 技術方式(段階導入)
### DF-1: 静的ローカル版(即日〜1日で実装可・承認後)
- `scripts/office/build_office.py`: wrangler経由でD1を読み→自己完結HTML(CSSピクセル部屋・外部依存ゼロ)を `local/office/index.html` に生成→ブラウザで開く
- 更新: 手動コマンド or セッション開始時に司令塔が再生成。**公開しない(ローカル専用)**=原典の「公開するのは怖い・PCだけでいい」に従う
### DF-2: 自動更新版(DF-1の使用感を見て)
- タスクスケジューラで30分毎に再生成+ブラウザ自動リロード / Discordの#報告-通知へ日次サマリ画像(任意)
### DF-3: キャラ完成版(S7と同時)
- 立ち絵・部屋アート・歩行アニメ・吹き出しの演出強化。スマホ対応(Mobile View)はこの段階で再検討

## 5. 作らないもの(明確化)
- リアルタイム常駐サーバ(スポーン型組織に常時接続は不要=静的生成で十分)
- 「今まさに働いている」風の嘘アニメーション(タスクが無い時は正直に待機)
- 公開URL(内部情報のため)
