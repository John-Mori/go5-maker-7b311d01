## §1 現状(コードから確認できる事実)

### 列定義の正本

```
CANONICAL_HEADERS = HEADERS40(33列) + FANZA_HEADERS(6列) + EXTRA_HEADERS(11列) = 計50列
```

| グループ | 列数 | 代表列 |
|---|---|---|
| HEADERS40 | 33 | post_id / 投稿日時 / 題名(コメント) / post_uri / YouTube動画URL / 短縮URL 等 |

### ヘッダー移行の状態

GAS 2026-07-22B の deploy 後処理(`admin_setup` → `migrate_headers` → `reorder_headers`)が:

- **月詠み**: `already_up_to_date` ← 全50列が既に存在

→ **列の種類と順番はコード上は両シート同一**。ヘッダー不一致は現時点では起きていない。

## §2 実測で確認すべき事項

以下はコードだけでは分からない。`sheet_audit` の実行が必要。

### 2-1 実際のヘッダー一覧と CANONICAL との差分

- `sheet_audit` の `headers` フィールドで実列一覧が取れる
- `missingFromCanonical` = CANONICAL にあるが実シートに無い列
- `extraColumns` = 実シートにあるが CANONICAL に無い列(手動追加等)

**実行コマンド**: `?action=sheet_audit`

### 2-2 行数と分類

| 分類 | 定義 |
|---|---|
| complete | postUri(at://) + YouTube動画URL 両方あり |
| no_yt | postUri あり・YouTube URL なし(Bluesky投稿済みだがYT未登録) |
| no_uri | postUri なし・YouTube URL あり(旧式記録・Bluesky投稿前の行) |
| minimal | postUri / YouTube URL どちらも無し |
| empty | post_id が空(clearContent()で消された行の跡) |

### 2-3 重複 postUri の有無

同一 postUri が2行以上ある場合 = **統合の候補**。
`dupDetail` に最大10件が返される。

### 2-4 データ密度の差(月詠み vs 宵桜艶帖)

両シートの `complete` 件数の比が、実運用の差を示す。
宵桜艶帖のほうが運用開始が遅い場合、旧形式の `no_uri` 行が多い可能性がある。
