# 【Deep Research 依頼】無料で「短い・計測できる・消えない」短縮URLを実現する方法を探す

> このドキュメントは、あるWebアプリの「短縮URL」機構の課題を整理し、解決策を深く調査してもらうための依頼文です。前提知識ゼロでも理解できるよう自己完結で書いています。**最終的に「具体的に実行可能な解決策」と「比較表」「推奨案」を提示してください。**

---

## 1. システムの背景（何をしているアプリか）

- 個人運用の **完全クライアントサイド（ブラウザだけで動く）Webアプリ**。GitHub Pages でホスティング（静的サイト）。サーバーは持たない。
- 用途：写真から短い縦動画を作り、**Bluesky** に投稿し、**YouTube Shorts** にもアップする運用支援ツール。Bluesky のアカウントは2つ運用。
- 投稿した **Bluesky投稿のURLを短縮**して、**YouTube動画の概要欄**に貼っている。「動画を見た人がどれだけそのリンクを踏んだか（クリック数）」を計測して、Googleスプレッドシートの分析表に記録したい。
- スプレッドシート連携は **Google Apps Script (GAS)** 経由。GASが毎時、短縮URLのクリック数を取得して `短縮URLクリック数` 列に書き込む。

## 2. 現状の短縮URL構成（技術詳細）

- 一次：**自前の Cloudflare Worker**（無料枠）を短縮サービスとして自作。
  - URL例：`https://r2.trustsignalbot.workers.dev/qr7K2`
  - 仕組み：`POST /api/shorten` でコード発行＋KVに `code→URL` を保存。`GET /:code` で **302リダイレクト**＋KVでクリック数を+1。`GET /api/stats?code=&secret=` でクリック数を返す。
  - 中間ページ（プレビュー/広告）が出ない 302 即リダイレクト。クリック計測はKVカウンタ。
  - 共有シークレット（ソフト鍵）でAPIを保護。発行元Originも制限。
- フォールバック：自前ワーカーが失敗したら **da.gd → TinyURL**（外部無料短縮）に自動で切替（ただし**外部短縮はクリック計測できない**）。
- Cloudflareアカウントは保有（`*.trustsignalbot.workers.dev` の無料サブドメイン）。wrangler CLI で worker/KV をデプロイできる程度の技術力はある。低メンテナンスを希望。

## 3. 解決したい課題（4条件を同時に満たしたい）

短縮URLに求める要素は次の4つ。**この4つを同時に満たす方法を探している。**

| # | 要素 | 説明 |
|---|------|------|
| ① | **短い** | `x.gd/abc` のように短いほど良い（理想は全体で10〜15文字程度） |
| ② | **無料** | 継続課金しない（年額・月額を払いたくない） |
| ③ | **クリック計測** | 各短縮URLのクリック数を**プログラムから取得**できる（API等。スプレッドシートに自動記録するため） |
| ④ | **消えない（リンクが死なない）** | 一度発行した短縮URLが将来も生き続ける。特に「自分が支払いを止めた／サービスが終了した」で**過去のリンクが全滅するのを避けたい** |

### 現状のトレードオフ（ここが核心の悩み）

| 選択肢 | ①短い | ②無料 | ③計測 | ④消えない |
|---|---|---|---|---|
| 自前Cloudflare Worker（現状 r2） | △ 長い(約42字) | ✅ | ✅ | ✅ |
| 外部無料短縮（x.gd / da.gd / TinyURL等） | ✅ | ✅ | ❌ | △ 先方依存 |
| 独自ドメイン（有料）＋自前Worker | ✅ | ❌ 年額 | ✅ | △ 払えば |

→ **現状、①と③④を両立できていない。** 自前ワーカーは「無料・計測・消えない」を満たすが、`xxxxx.workers.dev` が長い。外部短縮は短いが計測できない＆将来消えるリスク。

## 4. すでに検討して却下した案（同じ提案は不要）

- **独自ドメインを買う（例：`.link`/`.xyz` 年1,800円程度、`.gd` 年$30〜40）**：①③④は満たすが、**②無料に反する**＋「支払いを止めたら過去URLが全滅」という④の不安が残るため却下。
- **Cloudflareアカウントのworkers.devサブドメイン名を短く変更**：他の用途で使っている別のCloudflare Worker（FANZA情報取得用・Googleドライブ保存用）のURLまで一斉に変わって壊れるため却下。
- **Bitly**：無料枠の制限が厳しく、過去に全廃済み。
- **外部短縮のダッシュボードをスクレイピングしてクリック数を取得**：先方のログイン情報の保存が必要・CORSのため中継サーバーが必要・先方のUI変更で壊れる・規約グレー、という理由で非推奨と判断。

## 5. 制約・前提条件（解決策が満たすべき条件）

- アプリは**ブラウザ完結（静的）**。サーバー常駐は不可。**CORS**の制約あり（外部APIをブラウザから直接叩けないことが多い）。ただし **Cloudflare Worker を中継/プロキシとして自作するのは可能**。
- **Cloudflareアカウントあり**（無料プラン）。Worker・KV・（必要なら）カスタムドメイン/カスタムホスト名の設定が可能。
- 月額/年額の**継続課金は不可**（無料 or 完全買い切りのみ）。
- **低メンテ**希望（頻繁に手当てが要る方式は避けたい）。
- クリック計測は**正確である必要はなく概算でOK**（ユニークでなく総クリックで可）。
- 短縮URLの宛先は主に **bsky.app**（Bluesky投稿）と **youtube.com/youtu.be**。

## 6. 調査してほしいこと（具体的な問い）

次の観点で、**2026年時点で実在する具体的サービス名・無料枠の条件・URL構造・API有無**まで踏み込んで調べてください。「たぶんできる」ではなく**実際に可能か裏取り**してほしい。

1. **無料で、短い独自っぽいドメインを“買わずに”入手できる方法はあるか？**
   - 例として「**eu.org**（無料サブドメイン・自前ネームサーバー指定可）」「**js.org**」「**is-a.dev**」等の無料ドメイン/サブドメイン提供サービスを、**Cloudflareに登録して自前Workerのカスタムドメインに割り当て**できるか？（＝無料・短め・消えない・計測ありを同時に狙えるか）。実現可否・申請条件・所要時間・恒久性（消えないか）を検証してほしい。
   - これらの無料ドメインの**実際の長さ**（例 `name.eu.org` は何文字か）と、短縮URL用途での実用性。

2. **無料枠でクリック分析APIを提供している短縮サービスは存在するか？**（短いURL＋API経由でクリック数取得が両立できるもの）
   - 候補例：Cutt.ly / Short.io / Switchy / T.LY / Kutt(セルフホスト) / YOURLS(セルフホスト) など。**無料枠の上限・APIでクリック数が取れるか・URLの長さ・ブランドドメインの要否**を比較。

3. **自前ホスト型OSS短縮ツール**（YOURLS, Kutt, Shlink 等）を、**無料インフラ（Cloudflare Workers/Pages, Deno Deploy, Vercel無料枠等）**に載せて、**無料の短いドメイン**と組み合わせる構成は現実的か？ 恒久性・メンテ負荷・無料枠の制限を評価。

4. **リダイレクト・チェーン**（外部の短いURL → 自前Workerの計測URL → 最終URL）で「短い見た目＋自前で計測」を両立する方式の妥当性。デメリット（外部依存で消えるリスク・プレビューページ・二重短縮の手間）も含めて評価。

5. その他、**①短い ②無料 ③計測 ④消えない を可能な限り同時に満たす**、上記で挙げていない発想・サービス・アーキテクチャがあれば提案。

## 7. 出力してほしい形式

- 各解決策について **①〜④の充足度を○△×で表にする**（現状の表に追記する形）。
- **無料枠の具体的条件**（上限・制限・将来課金化リスク）と**恒久性の評価**を明記。
- **実装手順の概要**（誰が・何を・どの順で設定するか）。
- 最後に **「この運用（個人・無料・低メンテ・Cloudflare保有・クリック計測したい）に最も適した推奨案」を1つ**、理由付きで提示。
- 不確実な点は「要検証」と明示し、断定しないこと。

---

### 補足：現状の到達点
- 自前Cloudflare Worker（`r2`）でクリック計測は**動作確認済み**（短縮→302→クリック数取得まで成功）。唯一の不満は **URLが長い**こと。
- したがって理想は「**今のr2の計測の仕組みは活かしつつ、URLだけを無料で短く・恒久的にする**」方向。これを実現する具体策を最優先で探してほしい。
