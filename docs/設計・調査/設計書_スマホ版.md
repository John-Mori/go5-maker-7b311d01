# 設計書：5秒動画 自動生成ツール【iPhone版（スマホ・Webアプリ）】

> 本書は **スマホ（iPhone）版**の設計・保守ドキュメントです。PC版は親フォルダの `設計書_PC版.md` を参照。
> 目的：現状の実装を正確に記録し、**今後のメンテナンス・改良を迷わず行える**ようにする。
> 方針：**PCもサーバーも使わず、合成をスマホのブラウザ内（JavaScript/Canvas）で実行**する完全クライアントサイド方式。

---

## 0. 概要・ゴール

iPhone等のブラウザだけで、写真とテキストから **5秒・縦型(9:16)** の動画を作る。
- **PC不要・常時稼働サーバー不要**（端末内で完結＝「スマホ単体で動く」）。
- 仕上がり（背景・文字3段・アイコン・前景フェード・1行コメント・ファイル名）は **PC版 `composite.py` と統一**。
- 写真・生成動画は**どこにもアップロードされない**（端末内処理。プライバシー安全）。

### 公開状況
- ホスティング：**GitHub Pages**（無料・常時公開）。
- リポジトリ：`github.com/John-Mori/go5-maker-7b311d01`（推測されにくい名前＝URLを知る人だけ）。
- **公開URL：https://john-mori.github.io/go5-maker-7b311d01/**

## 1. なぜブラウザ内合成なのか（PC版との根本的な違い）

動画合成は本来 numpy/ffmpeg を使う重い処理だが、要望は「スマホ単体」。
そこで **Canvas 2D で1フレームずつ描画 → MediaRecorder で録画して mp4 化** する方式にした。
これによりサーバーが不要になり、**静的ファイルを置くだけ**で動く。

| 項目 | PC版 | スマホ版 |
|---|---|---|
| 合成 | Python `composite.py`（numpy/ffmpeg） | **ブラウザ内 Canvas＋MediaRecorder** |
| 画面 | Tkinter | HTML（Safari/Chrome） |
| 画像選択 | 参照/ドラッグ&ドロップ | `<input type=file accept=image/*>`（iOSの写真ピッカー/カメラ） |
| 文字入力 | tk.Entry（IME注意） | HTML入力（iOS純正IMEがそのまま効く＝問題なし） |
| 出力 | outputs/ に保存 | `<video>`表示 → 共有シートで保存 |
| 配布 | .bat 起動 | URLを開くだけ |

## 3. 仕上がり仕様（PC版と統一・`app.js` 冒頭の定数）

```js
W=1080, H=1920           // 基準フレーム（9:16・唯一の基準座標系。出力動画もこの解像度）
DURATION=5, FPS=30
REVEAL_START=0.5, REVEAL_DUR=2.0
FG_MAX_RATIO=0.92, FG_ZOOM=0.04, FG_CENTER_Y=0.55
DEFAULT_DETAIL="作品の詳細は右上の：から説明"
const U = (v) => v * H / 1280  // 旧1280基準の絶対px定数を基準フレーム高さに比率換算
```
- **座標系は基準フレーム(W×H)への比率で統一**。位置・フォントは `H×係数`／`W×係数`、
  余白・行間・縁取り下限などの旧絶対px定数は `U()` で基準フレーム高さに紐づく。
  → `W,H` を変えるだけで出力解像度が変わり、レイアウトは比率を保って追従する（`index.html` の
  `<canvas width height>` も `W,H` と一致させること）。
- **プレビューも書き出しも同一 Canvas(=基準フレーム解像度) に同じ式で描画**するため、
  画面幅が変わってもプレビュー同士・プレビューと出力が一致する（CSSは9:16を一様縮小表示するだけ）。
- フォント確定後(`document.fonts.ready`)に再描画し、フォールバックフォント計測由来のズレを防止。
- テキスト3段（PC版と同じ）：
  - 作者（`H×0.025`・**先頭に「作者：」を自動付与**）／誘導文（`H×0.027`・**「：」→⋮、「説明」→≡説明**）／
    コメント（`H×0.048`・**必ず1行**）。
  - コメントは `fitOneLine()` で横幅に収まらなければ自動縮小（最小 `U(14)`）。**ファイル名にもなる**。
- 前景：`REVEAL_START`から`REVEAL_DUR`でフェードイン（微ズーム）、最大占有92%、中心y=0.55。
- **構成全体の縦位置オフセット `vOffset`**（基準フレーム高さ比・既定 `VOFF_DEFAULT=0.02`／上限 `VOFF_MAX=0.05`）。
  上部テキスト開始Y（`H×(0.020+vOffset)`）と前景中心Y（`H×(FG_CENTER_Y+vOffset)`）に同じだけ加算し、
  構成全体を一体で下方向へ微調整する（相対関係＝中央揃え・行間は不変）。`index.html` のスライダー `#voff`
  で調整し `localStorage["v_offset"]` に保存・復元。比率ベースのためPC/スマホ/書き出しで一致。

> PC版（`composite.py` / `config/jobs.json`）の値を変えたら、ここの定数も合わせると見た目が揃う。
> 解像度を変えたいときは `W,H`（と `index.html` の canvas 属性）だけを変える＝比率は全て自動追従。

## 4. app.js の主要部（合成と録画）

| 関数 | 役割 |
|---|---|
| `drawFrame(t)` | 時刻tの1フレームを描画（背景cover → 前景フェード → テキスト） |
| `drawText / drawBlock` | 文字段の描画（中央・帯・白文字＋黒縁） |
| `drawDetail` | 2段目のアイコン入りインライン描画（⋮ / ≡） |
| `_draw kebab / hamburger` 系 | アイコン描画 |
| `fitOneLine` | 3段目コメントを1行に収める（自動縮小） |
| `preview()` | 完全表示状態の1枚をCanvasに描画 |
| `make()` | 背景再生→`captureStream(30)`→`MediaRecorder`で5秒録画→Blob→`<video>`表示 |
| 保存 | `navigator.share`（共有シート）／ダウンロード |

- 動画形式：iOS Safari は **mp4(H.264)**、非対応環境は **webm** にフォールバック（`pickMime`）。
- フォント：Google Fonts の **Noto Sans JP**。Canvas描画前に `document.fonts.load` で確実化。
- 背景：同梱 `assets/bg_main.mp4` を `<video muted loop playsinline>` で再生し、Canvasに描画。
- プレビュー更新は **入力確定（blur/change）／更新ボタン** 時（iOSでも素直に動く）。

## 5. カスタマイズ／改良ガイド

| やりたいこと | 触る場所 |
|---|---|
| 文字サイズ・位置・帯・アイコン | `app.js` の `drawText` / `drawBlock` / `drawDetail` / `_draw*` と定数 |
| 前景の大きさ・位置・フェード | `app.js` 冒頭の `FG_*` / `REVEAL_*` 定数（PC版と合わせる） |
| 背景を差し替える | `assets/bg_main.mp4` を別の9:16動画に置換（Web用に軽くする：`ffmpeg -vf scale=720:1280 -crf 28`） |
| 背景を複数選べるようにする | `index.html` に選択UI、`app.js` で `<video>` の src を切替 |
| 出力解像度を上げる | `W,H` を 1080×1920 に（モバイル負荷とのトレードオフ） |
| 出力ファイル名規則 | `app.js` の `sanitize()` / `lastName` |

## 7. アクセス制御・安全

- **端末内処理**のため写真・動画は外部送信されない（漏えいリスクが構造的に無い）。
- 公開URLは推測されにくいリポジトリ名にし、**リンクを広く共有しない**＝「知っている人だけ」。
- 完全クライアントサイドのため「サーバー側パスワード判定」は不可（鍵をJSに書いても公開されるため無意味）。
  厳密な認証が要る場合は Cloudflare Access 等ホスティング側の制御を別途使う（任意・上級）。
- 前景画像の内容判定はしない（権利的に使用可・未成年の性的描写を含まない素材を利用者が用意）。

---

## 8. 既知の制約

- iOSのバージョン/ブラウザにより MediaRecorder の対応や出力形式（mp4/webm）が異なる。iOS 15以降のSafari推奨。
- 内部解像度720pは性能優先。高画質化は `W,H` を上げるが録画負荷が増える。
- 背景mp4のサイズが大きいと初回読み込みが重い（Web軽量版を推奨）。
