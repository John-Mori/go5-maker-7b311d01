## 1. 改修の目的

既存の「5秒動画メーカー（スマホ版・完全クライアントサイド）」に、
**FANZAアフィリエイトリンク生成機能**を1画面として追加する。

- 作品URLを貼ると **作品ID（cid）** と **アフィリエイトリンク** を自動生成。
- 抽出は全て文字列処理。手作業コピペによる取り違え・計測パラメータ混入を構造的に排除。
- 既存の動画作成機能には**一切手を入れない**（`app.js` は無変更で流用）。

## 2. 統合方針（決定事項）

| 論点 | 決定 | 理由 |
|---|---|---|
| 配置 | **ヘッダー直下にタブバー**（🎬 動画作成 / 🔗 アフィリンク）で2画面切替 | 既存はナビ無しの単一ページ。最小改修で1機能ページを増やせる |
| 切替方式 | 2つの `<main>` 相当セクションを JS で `hidden` トグル | SPA フレームワーク不使用（既存スタックは素のJS） |
| コアロジック | `affiliate-core.js` に純粋関数 `buildAffiliateLink()` を分離 | UI非依存・Node でもブラウザでも動く形（テスト可能） |
| UI配線 | `affiliate.js` に分離 | `app.js`（動画）と責務を混ぜない |
| デザイン | 既存 `style.css` のデザイントークン（`--bg --card --ink --sub --accent --line`、`.card` `.field-label` `button.primary` `.ghost`）に準拠 | 設計書の指示どおり |
| 永続化 | `localStorage` キー **`fanza_af_id`** | ブラウザベースのため。URL・ログに残さない |
| テスト | `tests/test_affiliate.js`（Node 実行）で T-1〜T-4＋エッジケース | 受け入れ条件の自動検証 |

## 4. 画面設計

### 4.1 タブバー（`index.html` ヘッダー直下）

```html
<nav class="tabbar">
  <button id="tabMovie" class="tab active">🎬 動画作成</button>
  <button id="tabAffi"  class="tab">🔗 アフィリンク</button>
</nav>
```
- `#pageMovie`（既存 `<main>` の中身全部）と `#pageAffi`（新画面）を切替。
- 切替は `hidden` 属性のトグル＋タブの `.active` クラス付替えのみ。状態保存はしない（リロードで動画作成タブに戻る）。
- 動画作成側の DOM 構造・id は**一切変更しない**（`app.js` が `getElementById` で参照しているため）。

### 4.2 アフィリンク画面（`#pageAffi`）

```
[card] ① あなたのアフィID
       <input id="afId" type="text">         ← localStorage 自動復元
       （未入力時の警告文 .affi-warn）
[card] ② 作品URL（1行に1URL・複数可）
       <textarea id="affiUrls" rows="4">     ← input イベントでリアルタイム生成
[card] 生成結果 <div id="affiResults">
       URL 1行ごとに結果カード：
         ・作品ID: <code>d_748504</code> ［IDコピー］
         ・リンク: <code class="affi-link">https://al.fanza...</code> ［リンクをコピー］
         ・エラー時: 理由表示（結果は出さない）
```

- **リアルタイム生成**（F-3）：`afId` / `affiUrls` の `input` イベントで即再描画。ボタン不要。
- **複数行対応**（F-2）：textarea の非空行それぞれを `buildAffiliateLink()` に通し、行ごとに結果カードを出す。
- **コピー**（F-5/F-6）：`navigator.clipboard.writeText` を使い、失敗時は一時 textarea＋`document.execCommand("copy")` にフォールバック（旧iOS対策）。コピー成功時はボタンラベルを「✓ コピーしました」に2秒間変更。
- **警告**（F-7）：アフィID未入力時は入力欄下に警告を出しつつ、リンクは `af_id=【アフィID】` で構造プレビューを生成する。
- **エラー**（F-8）：`no_cid` →「cid が見つかりません」、`bad_url` →「URLが不正です（http(s):// で始まる必要があります）」。該当行の結果は出さない。
- **空入力**：何も表示しない（エラーにしない）。

## 5. コアロジック仕様（厳守・元設計書 第5章のまま）

`affiliate-core.js` に以下を**そのまま**実装する。挙動を変えない。

1. **cid 抽出**：`/cid=([^/?&\s]+)/`。マッチしなければ `{ok:false, error:'no_cid'}`。
2. **URLクリーニング**：`?` 以降を除去 → `^https?:\/\//i` でなければ `{ok:false, error:'bad_url'}` → 末尾 `/` を補完。
3. **lurl 生成**：クリーニング済みURLを `encodeURIComponent`。
4. **組み立て**：`https://al.fanza.co.jp/?lurl={ENCODED_URL}&af_id={AF_ID}&ch=toolbar&ch_id=link`（`&ch=toolbar&ch_id=link` は固定）。
   `AF_ID` は空なら `【アフィID】`。

```javascript
function buildAffiliateLink(rawUrl, afId) {
  const raw = (rawUrl || '').trim();
  if (!raw) return { ok: false, error: 'empty' };
  const m = raw.match(/cid=([^/?&\s]+)/);
  if (!m) return { ok: false, error: 'no_cid' };
  const cid = m[1];
  let clean = raw.split('?')[0].trim();
  if (!/^https?:\/\//i.test(clean)) return { ok: false, error: 'bad_url' };
  if (!clean.endsWith('/')) clean += '/';
  const lurl = encodeURIComponent(clean);
  const af = (afId && afId.trim()) ? afId.trim() : '【アフィID】';
  const link = `https://al.fanza.co.jp/?lurl=${lurl}&af_id=${af}&ch=toolbar&ch_id=link`;
  return { ok: true, cid, link };
}
```

- `/dc/doujin/` をハードコードしない（URL全体エンコード方式のため他カテゴリでも動く）。
- エクスポート形態：`window.buildAffiliateLink = buildAffiliateLink;`（ブラウザ）＋
  `if (typeof module !== 'undefined' && module.exports) module.exports = { buildAffiliateLink };`（Node テスト用）。

## 6. アフィIDの永続化

- 保存：`afId` の `input` イベントで `localStorage.setItem('fanza_af_id', value)`。
- 復元：`affiliate.js` 初期化時に `localStorage.getItem('fanza_af_id')` で入力欄へ反映。
- URLパラメータ・コンソールログに af_id を出さない。

## 7. style.css 追記方針

既存トークンのみ使用。追記するのは概ね：
- `.tabbar`（横並び2分割）/ `.tab` / `.tab.active`（`--accent` 下線 or 背景）
- `textarea`（`input[type=text]` と同スタイル＋`resize:vertical`）
- `.affi-result`（結果カード内の小ブロック）/ `.affi-code`（リンク表示：`word-break:break-all`・等幅可）
- `.affi-warn`（警告色は `.notice` と同系 `#b89b6b`）/ `.affi-error`（`--accent2`）
- コピー用小ボタン `.copy-btn`（`.ghost` 流用可・タップ領域 44px 以上確保）

## 13. 追加改修（プレビュー描画の改善・基準座標系の統一）

アフィリンク統合後に実施したプレビュー/書き出しの改善。`app.js` と `index.html` を変更（アフィリンク機能には影響なし）。

### 13.1 検証で判明した事実（不具合仮説の検証結果）
- 「テキスト座標が画面px(vw/vh)基準で出力フレームに紐づいていない」という仮説は **誤り**。
  座標は元々固定フレーム比率で計算され、書き出しも同一Canvasを `captureStream` で録画していたため、
  画面幅375px/320pxで内部描画ピクセルは完全一致（実測）。プレビュー表示も常に9:16を維持。
- 真に不完全だったのは **余白・行間・縁取り下限などに混在した「フレーム解像度に紐づかない絶対px定数」**。

### 13.2 実施内容
1. **基準フレームを 1080×1920 に統一**（`W,H` と `index.html` の `<canvas>` 属性）。唯一の基準座標系。
2. **絶対px定数を `U(v)=v*H/1280` で基準フレームに比率換算**。位置・フォントは `H×係数`／`W×係数`。
   → `W,H` を変えるだけで出力解像度が変わり、レイアウトは比率を保って追従。
   → 1.5倍解像度で相対位置・サイズが旧フレームと一致することを実測確認（誘導文 yTop6.0%/w71%・
     コメント yTop10.2%/w49% が一致）。
3. **フォント確定後（`document.fonts.ready`）に再描画**。フォールバックフォント計測由来の
   プレビュー↔書き出しのズレを防止。
4. 作者行は **先頭に「作者：」を自動付与**（既に「作者」始まりなら二重付与しない）。
5. テキスト帯を縁取り幅ぶん拡張＋不透明度引き上げ（明るい動画背景でも文字が帯からはみ出ない）。
