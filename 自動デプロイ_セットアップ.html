⚙️ 自動デプロイ セットアップ

これ1回で、以後ずっと「PC不要・操作不要」でCloudflare自動デプロイ

所要 約5分・全部スマホのブラウザでOK

これをやると： 私(Claude)がworkerのコードを直して main に push するだけで、 GitHubが自動で Cloudflare にデプロイしてくれます。外出先でも、あなたが何もしなくてもOK。 設定ファイル（GitHub Actions）はもう入れてあります。あとは“鍵”を2つ登録するだけ。

パートA：Cloudflareの鍵（APIトークン）を作る

1

APIトークン作成画面へ

Cloudflare APIトークン画面を開く

ログインを求められたらログイン。

2

テンプレートを使う

  • Create Token（トークンを作成）を押す
  • テンプレ一覧から 「Edit Cloudflare Workers」 の Use template
  • 設定はそのままでOK（自分のアカウントが選ばれているか確認）→ 下の Continue to summary → Create Token
3

トークンをコピー

表示された長い文字列をコピー。

注意：このトークンはこの画面でしか見られません。コピーして、次のパートCで貼ります（メモアプリ等に一時保存可・使ったら消す）。

パートB：Account ID を控える

4

Account ID をコピー

Workers & Pages を開く

その画面の右側（または下の方）に 「Account ID」 があります。コピー。 （任意のドメインの「概要」右下にも出ます）

パートC：GitHubに鍵を2つ登録

5

GitHubのSecrets画面へ

GitHub Secrets画面を開く

GitHubにログイン後、この画面（Settings → Secrets and variables → Actions）が開きます。

6

2つ登録する

New repository secret を押して、名前は下の通り正確に（コピペ推奨）：

① 1つ目：

Name: CLOUDFLARE_API_TOKEN
Secret: （パートAでコピーしたトークンを貼る）

→ Add secret。もう一度 New repository secret で ②：

Name: CLOUDFLARE_ACCOUNT_ID
Secret: （パートBのAccount IDを貼る）

→ Add secret。

完了！ 画面に CLOUDFLARE_API_TOKEN と CLOUDFLARE_ACCOUNT_ID の2つが並んでいればOK。

これで何が起きるか

  • 今後、私がworkerを直して push するたび、GitHubが自動でCloudflareへデプロイ（あなたは何もしない）。
  • 手動で動かしたい時：GitHubの Actions タブ → 対象のワークフロー → Run workflow（スマホでも押せます）。
  • 鍵はGitHubが暗号化保管し、ログにも出ません。トークンは「Workers編集」だけの最小権限。
登録できたら、チャットで「secrets登録した」と教えてください。 今回の drive-worker の変更（元写真＋プレビュー2枚保存／ゴミ箱の連番除外）を、私の方からその場でデプロイ実行します。以後は完全自動です。

どこかで画面が違う／詰まったら、何が見えているかを送ってください。1タップずつ伴走します。
