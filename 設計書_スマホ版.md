# 設計書：5秒動画ツール スマホ（iPhone）対応版 ― クライアントサイド方式

> 目的：PCで使っている「5秒動画 自動生成ツール」を、**iPhoneのブラウザだけで完結**して作れるようにする。
> 方針：**PCもサーバーも使わず、合成をスマホのブラウザ内（JavaScript/Canvas）で実行**する。
> 公開は GitHub Pages（無料・静的ホスティング）に置くだけ。**URLを知っている人だけ**が使う。

---

## 0. ゴールと前提

- iPhoneのSafariで **URLを開く → 写真を選ぶ → 文字を入れる → 「動画を作成」 → 端末に保存**。
- **PC不要・常時稼働サーバー不要**。すべて端末内で完結（＝「スマホ単体で動く」）。
- 仕上がり（背景・文字3段・前景画像・浮かび上がり・アイコン）はデスクトップ版と同じ見た目。
- 写真や生成した動画は**どこにもアップロードされない**（端末内処理）。プライバシー面でも安全。

### なぜ「ブラウザ内で合成」なのか
当初はサーバーで合成する案だったが、要望は「スマホ単体で動く」。
そこで合成を **HTML5 Canvas で1フレームずつ描画 → MediaRecorder で録画して mp4 化** する方式にした。
これによりPCもクラウドサーバーも不要になり、静的ファイル（HTML/JS/CSS＋背景mp4）を置くだけで動く。

```
[iPhone Safari]
  写真選択(端末内) → Canvasで合成描画 → MediaRecorderで録画 → mp4をその場で保存/共有
  （通信は最初にページと背景mp4を読むだけ。以降サーバーは不要）
```

## 1. 現行（デスクトップ版）との違いの整理 ★重要

| 項目 | デスクトップ版（現行） | スマホ版（本設計） |
|---|---|---|
| 画面 | Tkinter のウィンドウ | **Webページ**（Safari/Chrome） |
| 画像の選択 | 「参照…」/ ドラッグ&ドロップ | `<input type="file" accept="image/*">`。**iOSで「写真を選ぶ／撮る」が自動で出る** |
| 文字入力 | Tkinter Entry | HTML入力欄（iOSのIME・変換がそのまま使える） |
| プレビュー | その場で1枚合成 | **Canvasに1枚合成して表示**（入力確定で更新＝IMEを妨げない） |
| 合成の実行 | PC上で numpy/ffmpeg | **スマホのブラウザ内**（Canvas＋MediaRecorder） |
| 出力 | `outputs/` に保存 | **その場で `<video>` 表示 → 「保存/共有」で写真/ファイルへ** |
| 起動 | `.bat` ダブルクリック | URLを開くだけ |
| ファイル名 | コメント文字列 | 同じ（コメント＝ファイル名） |
| 背景 | プルダウンで複数 | まずは1本同梱（あとで追加可） |

### スマホならではの考慮点（実装済み）
- **写真アクセス**：`accept="image/*"` でiOSが写真ピッカー/カメラを提示。HEICはiOSが自動でJPEG化して渡すため通常通り読める。
- **縦持ち1カラムUI**。
- **処理待ち**：作成時は「作成中…」表示＋ボタン無効化（5秒録画）。
- **保存**：`navigator.share`（共有シート）で写真に保存。非対応端末はダウンロード/長押し保存。
- **IME対策**：プレビュー更新は入力確定（blur/Enter）と「更新」ボタンのみ。毎キーでは更新しない。
- **動画形式**：iOS Safariの MediaRecorder は **mp4(H.264)** を出力。非対応環境では webm にフォールバック。

## 2. アクセス制御（URLを知っている人だけ）

- **そもそも端末内処理**なので、写真・動画は外部に送信されない（漏えいリスクが構造的に無い）。
- 公開URLは **推測されにくいリポジトリ名**にし、リンクを広く共有しない＝「知っている人だけ」。
- ※ 完全クライアントサイドのため「サーバー側でパスワード判定」はできない（鍵をJSに書いても公開されるため無意味）。
  どうしても認証が要る場合は、Cloudflare Access 等のホスティング側アクセス制御を別途使う（任意・上級）。

## 3. ホスティング（公開方法）

| 方式 | 内容 | 補足 |
|---|---|---|
| **GitHub Pages（推奨）** | リポジトリに静的ファイルを置き、Pagesを有効化 | 無料・常時公開・GitHubアカウントのみ。`username.github.io/リポジトリ名` のURL |
| Cloudflare Pages / Netlify | 同じ静的ファイルをデプロイ | ランダムなサブドメイン。アクセス制御(Access)も付けられる（任意） |

- 配布物（＝`スマホ用/` の中身）はそのまま静的サイト。サーバープロセスは無し。
- 手順は `README.md` を参照（GitHubのWeb画面にドラッグ&ドロップでアップロード→Pages有効化）。

## 4. 技術スタック

| 役割 | 採用 |
|---|---|
| 画面 | 素のHTML/CSS/JS（フレームワーク無し・iOS Safari互換重視） |
| 合成 | **Canvas 2D**（背景フレーム＋前景＋テキスト/アイコンを描画） |
| 動画化 | **MediaRecorder**（canvas.captureStream を5秒録画 → mp4/webm） |
| 日本語フォント | Google Fonts の **Noto Sans JP**（CDN）。Canvas描画前に `document.fonts.load` で確実化 |
| 背景 | 同梱の `assets/bg_main.mp4`（Web用に720×1280へ軽量化） |
| 保存 | Web Share API（`navigator.share`）＋ダウンロード |

## 6. 仕上がり仕様（デスクトップ版 composite.py と統一）

- 解像度：内部 720×1280（9:16）／長さ 5秒／30fps。
- テキスト（上から）：作者 → 誘導文（「：」を⋮、「説明」を≡説明に置換）→ コメント（大）。白文字＋黒縁＋半透明帯。
- 前景：`reveal_start=0.5s` から `reveal_dur=2.0s` でフェードイン（微ズーム）。最大占有 92%、中心 y=0.55。
- ファイル名＝3段目コメント（使用不可文字は除去、空なら video）。

---

## 7. 実装順序（実施済み）

1. `index.html` / `style.css`（モバイルUI・写真選択・入力・プレビュー枠）。
2. `app.js`：Canvas合成（背景cover→前景フェード→テキスト/アイコン）。
3. プレビュー（完全表示の1枚）。
4. 録画（MediaRecorderで5秒→Blob→`<video>`表示）。
5. 保存/共有（Web Share API／ダウンロード）。
6. 背景mp4のWeb軽量化、Noto Sans JP読込。
7. `README.md`（GitHub Pages公開手順・iPhoneでの使い方）。

---

## 8. 受け入れ基準

- iPhoneのSafariで公開URLを開き、写真を選び、文字を入れて「動画を作成」を押すと、
  デスクトップ版と**同じ仕上がり**の5秒・9:16動画が生成され、端末に保存できる。
- PCもサーバーも不要（スマホ単体で完結）。写真・動画は外部送信されない。
- 日本語が文字化けしない。作成中の待ち表示・二重押下防止がある。

> 注意：前景画像の選定・内容はツールの責務外（権利的に使用可・未成年の性的描写を含まない・配信先規約に適合する素材を利用者が用意）。
