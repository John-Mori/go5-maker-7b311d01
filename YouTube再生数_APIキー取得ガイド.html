▶ YouTube再生数 APIキー取得ガイド

「検証タブ」で再生数を自動表示するための準備（1回だけ）

所要 約10分・全部スマホのブラウザでOK・無料

YouTubeの再生数は、Googleの「YouTube Data API」という公式の窓口からしか自動で取れません。その窓口を使うためのカギ（APIキー）を、Googleの管理画面で1回だけ作ります。

難しそうに見えますが、やることは ① プロジェクトを作る → ② 窓口をオンにする → ③ カギを作る → ④ アプリに貼る の4つだけ。下に画面そのままの順で書いてあります。迷ったらそのまま上から進めてください。

よくある疑問：「動画をアップしているアカウントじゃないとダメ？」
→ いいえ。どのGoogleアカウントで作ったキーでもOKです。再生数は公開情報なので、APIキーで取るのにチャンネルの持ち主である必要はありません（キーは「利用枠」を示すだけで、動画の所有権とは無関係です）。
  • ✅ 公開の動画 … どのアカウントのキーでも再生数が取れます。
  • ✅ 限定公開（リンクを知っている人のみ） … 動画URLが分かっていれば取れます。
  • ❌ 非公開（プライベート） … これだけは持ち主本人のログイン認証が必要で、APIキーでは取れません（今回の仕組みは非対応）。
※同じアカウントに揃える必要はありません。揃えると「管理画面が分かりやすい」程度のメリットだけです。スマホで複数のGoogleにログインしている場合は、各画面の右上の丸いアイコンからアカウントを切り替えられます。

① プロジェクトを作る

1

Google Cloud コンソールを開く

下のボタンを押すと、Googleの管理画面（Google Cloud コンソール）が開きます。

▶ プロジェクト作成ページを開く
  • 初めての場合：利用規約の同意画面が出ることがあります。国＝日本を選び、規約にチェック→同意して続行を押します。
  • 「無料トライアルを開始」「お支払い情報の登録」などが出ても、登録は不要です。今回使う機能は無料枠だけで足ります。閉じるか「スキップ」でOK。
2

プロジェクトの名前を入れて作成

「新しいプロジェクト」の画面が開きます。

  • プロジェクト名に、自分が分かる名前を入れます（例：go5-youtube）。何でもOK。
  • 「場所」「組織」はそのままでOK。
  • 作成 を押します。
  • 右上に「プロジェクトを作成しています…」と少し出ます。10〜20秒ほど待つと完成です。
大事：作成後、画面上部の真ん中あたりに出る「プロジェクト選択」が、いま作ったプロジェクト名になっているか確認してください。違っていたら、そこを押して今作ったものを選びます。（これ以降の作業が、このプロジェクトの中に保存されます）

② 窓口（YouTube Data API）をオンにする

3

「YouTube Data API v3」を有効にする

下のボタンで、YouTube再生数の窓口ページが直接開きます。

▶ YouTube Data API v3 を開く
  • 青い大きな 有効にする（英語なら ENABLE）ボタンを押します。
  • 数秒で「APIが有効になりました」という画面に変わればOK。
もしボタンが「管理」になっていたら？
それはすでに有効という意味です。そのまま次へ進んでください。
「プロジェクトを選択してください」と出たら？
①で作ったプロジェクトを選んでから、もう一度このボタンを開いてください。

③ カギ（APIキー）を作る

4

「認証情報」からAPIキーを作成

下のボタンで「認証情報」ページが開きます。

▶ 認証情報ページを開く
  • 画面上の ＋ 認証情報を作成（英語なら + CREATE CREDENTIALS）を押します。
  • 出てきたメニューから APIキー を選びます。
  • 「APIキーを作成しました」と表示され、AIzaSy… で始まる長い文字列が出ます。これがカギです。
  • コピー を押して、いったんスマホのメモやLINEの自分宛などに貼り付けておくと安全です。
カギの例（これはダミーです）
AIzaSyD-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
※あなたのカギは他人に見せないでください。下の④で「使えるサイトを制限」しておくと、万一見られても悪用されにくくなります。

④ （推奨）カギに鍵をかける＝このサイト専用にする

5

制限をかけて安全にする（2分）

作ったキーの行の右にある ⋮（または鉛筆✏️） →「キーを編集」を開くか、作成直後のポップアップの「キーを制限」から設定します。

(A) アプリケーションの制限

  • ウェブサイト（HTTP リファラー）を選びます。
  • 「ウェブサイトの制限」で 追加 を押し、次の2つを登録：
https://john-mori.github.io/*
https://john-mori.github.io/go5-maker-7b311d01/*

(B) API の制限

  • キーを制限 を選び、リストから YouTube Data API v3 だけにチェックを入れます。
  • 最後に 保存 を押します。
なぜやるの？ こうすると、このキーはあなたのサイトからしか、しかもYouTubeの再生数取得だけに使えなくなります。安心して使えます。
※保存後、反映に数分かかることがあります。すぐ動かない時は5分ほど待ってから試してください。

⑤ アプリにカギを貼る

6

「⚙️詳細設定」タブに貼り付け

  1. 5秒動画メーカーを開く（下のボタン）。
  2. 上のタブの ⚙️ 詳細設定 を押す。
  3. 「▶ YouTube再生数の取得（APIキー）」の入力欄に、③でコピーしたカギ（AIzaSy…）を貼り付ける。
  4. これで保存完了（自動保存・この端末内だけに保存されます）。
▶ 5秒動画メーカーを開く
7

検証タブで再生数を表示

  1. 🧪 検証 タブを開く。
  2. 一番上の「▶🔗 再生数・クリック数（投稿別）」の各行にある「YouTube URL」欄に、その投稿に対応する動画のURL（https://youtu.be/… や https://www.youtube.com/watch?v=…）を貼る。
  3. 🔄 再生数・クリック数を更新 を押す。
  4. ▶ の横に再生数、🔗 の横に短縮URLのクリック数が出ます。
ヒント：ウィザード（🪄）でYouTube URLを記録した投稿は、URL欄が自動で埋まります。過去の投稿は手で貼ればOKです。

❓ うまくいかない時（よくある原因）

症状原因と対処
再生数が「–」のままその行の「YouTube URL」がまだ空、またはURLの形式が違います。youtu.be/動画ID か watch?v=動画ID を貼ってください。
「API key not valid」キーの貼り間違い（前後に空白が入った等）。もう一度コピーして貼り直してください。
「API has not been used / disabled」②の「YouTube Data API v3 を有効にする」が済んでいません。手順③をやり直してください。
「referer / requests from referer ... are blocked」④のサイト制限のURLが違います。https://john-mori.github.io/* を登録、または一旦「アプリケーションの制限＝なし」にして動作確認してください。
「quota exceeded」1日の無料上限（後述）に達しました。翌日（太平洋時間の0時＝日本の夕方頃）にリセットされます。
💰 料金と上限について（読まなくてもOK）
  • YouTube Data API は無料で、1日あたり 10,000ユニット使えます。
  • このアプリの再生数取得（videos.list）は1回1ユニット（しかも複数動画をまとめて1回で取得）。普通に使う分には上限にまず届きません。
  • クレジットカード登録は不要です。
🔒 セキュリティ（カギの扱い）
  • カギはこの端末のブラウザ内だけに保存されます。リポジトリやサーバーには置きません。
  • 「⚙️詳細設定」→ 設定の書き出しで「鍵を除いて書き出し」を選べば、キーは書き出されません（共有・バックアップ時も安全）。
  • ④の制限をかけておけば、万一キーが見られても他サイトから悪用できません。
  • もしキーを作り直したくなったら、③のページで古いキーを削除し、新しく作って貼り替えるだけです。
それでも分からない時は
「③のカギを作る」までできれば、あとはアプリに貼るだけです。どの画面で詰まったか（ボタン名や英語表示）を教えてくれれば、その場面に合わせて案内します。
