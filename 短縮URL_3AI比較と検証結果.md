## 0. 結論（先に要点）

- **今夜、私が実際に構築・テストして“合格”したのは「da.gd チェーン方式」だけ**。他の有力案（pp.ua / is-a.dev + Cloudflare独自ドメイン）は、**あなたのSMS認証やGitHub PR・人手審査が必須で、私が寝ている間に自律実行できない**ため未実施（＝準備・設計まで）。
- **da.gd チェーンは実測で成立**：`da.gd/aytu`（16字）→ r2 → 最終URL、まで到達し、**中間のr2でクリック数が 0→2 に増加＝計測維持を確認**。
- ただし da.gd チェーンは「②無料・①短い・③計測」は満たすが、**④消えない と 評価（SNSでの評判）と 遅延 に弱点**がある。あなたの用途（匿名寄りのアダルト系アフィリ）では **プライバシー観点**も重要で、そこが3AIの評価に一部欠けていた。
- **最もクリーンな本命は「独自の無料短いドメイン + 既存r2にCustom Domain割り当て」**だが、これは**あなたの手動ワンステップ（pp.uaのSMS or is-a.devのPR）が必要**。→ 別mdの設計書に、すぐ着手できる手順を用意した。

### 1-3. is-a.dev（開発者向け無料サブドメイン・GitHub PR方式）

| AI | 評価 | 根拠 |
|---|---|---|
| ChatGPT | **不適** | NS委任は例外審査で、Cloudflareのような「DNSプロバイダ目的」の委任は規約で不可。新規30日未満も拒否。→ 自前ゾーン化できない。 |
| Claude | **不適（同上）** | NS委任禁止で「自前Worker+Custom Domain」用途に使えない。 |

**突き合わせ判断**：ChatGPT/Claudeは「NS委任不可＝終了」で切ったが、**GeminiはCloudflare for SaaSという“別の接続方式”で回避策を提示**しており、ここは**Geminiの方が一段深い**。ただし：
- **フォールバックzoneとして結局pp.ua等が1つ必要**（そのzoneはWHOIS公開でも“表に出さない”ので可、とGeminiは説明）。

- 手順が最も複雑（GitHub Fork→PR→マージ待ち→Custom Hostname→TXT検証→SSL更新の継続性が要検証）。

### 1-5. 自前ホストOSS（YOURLS / Kutt / Shlink / Sink）

- **YOURLS(PHP+MySQL)/Kutt(Node+PG+Redis)/Shlink(PHP+DB)**：3AI一致で「サーバーレス無料構成に載らない・DB常駐が要る・低メンテに反する」→ **不適**。しかも**現状のr2+KVが既にこの“自前ホスト”を最小構成で達成済み**（新規OSS導入の妥当性が低い）。
- **Sink（miantiao-me/Sink）**：**Geminiのみが発掘**した Cloudflare Workers/Pages ネイティブのOSS。D1(正)+KV(キャッシュ)のハイブリッドで、API計測・CSVエクスポートも完備。**ただしこれも「短さ」自体は解決しない**（workers.devに載せれば同じく長い）。Sinkの価値は“機能拡張(OGP等)”であって、今回の主目的（短く）とはズレる。→ 今回は不要、将来の機能拡張候補。

## 2. 私の自律テスト結果（da.gd チェーンを隔離環境で実測）

> 既存アプリのコードには一切触れず、r2ワーカーにテスト用の短縮を作って検証。実データ・スプレッドシートには無影響。

### 手順と結果

1. r2で最終URL(bsky.appテスト)を短縮 → `r2.../U2j84`（43字）
2. **is.gd / v.gd → 現在サーバー障害中**（`Error, database insert failed` を連続で返す＝当てにできない）
3. **da.gd → 成功**：`https://da.gd/aytu`（**16字**）
4. tinyurl（28字）・cleanuri（27字）も成功するが長い。spoo.me はCloudflareにフィッシング判定でNG。
5. **チェーン到達テスト**：`da.gd/aytu` を2回たどる → いずれも **最終HTTP 200・リダイレクト2回・最終URL=bsky に到達**
6. **計測テスト**：チェーン通過後、**r2のクリック数が 0 → 2 に増加＝中間ホップでの計測が成立** ✅
7. **遅延**：da.gdチェーン ≈ 1.70s / r2直 ≈ 1.37s（**差 ≈ +0.35s**。大半は最終bsky.appの読み込み時間で、リダイレクト追加分は小さい）

### 判定：**da.gd チェーンは技術的に合格**

| 条件 | 判定 | 補足 |
|---|---|---|
| ①短い | ○ | 16字（現状43字から大幅短縮） |
| ②無料 | ○ | da.gd無認証・無料 |
| ③計測 | ○ | **r2中間ホップで計測維持を実測確認** |
| ④消えない | △ | **印字されるのは da.gd。da.gdが死ねば印字リンクは死ぬ**（外部依存が1つ増える） |
| 遅延 | △ | +0.35s程度。概要欄リンクなら許容 |
| SNS評判 | △(要観察) | 共有短縮はSNSで評価が下がりうる（Gemini指摘）。**ただしあなたは既にda.gdを一部投稿で使用中＝現状比で新規悪化ではない** |
| プライバシー | ○ | 登録不要＝個人情報の露出ゼロ |

## 3. 3AIが十分触れていない「この運用特有」の観点（私の追加分析）

1. **プライバシー（匿名性）**：あなたの運用はアダルト系アフィリで匿名性が重要。
   - pp.ua＝**WHOISで実名/電話/住所が公開**（致命的）。is-a.dev＝GitHub名/メールのみ（可）。da.gdチェーン＝登録不要（最も安全）。
   - → **プライバシーだけを見るなら da.gdチェーン ＞ is-a.dev ＞ pp.ua**。
2. **規約リスク**：is-a.devは「非商用・開発関連」限定。**アダルトアフィリは規約抵触で剥奪の可能性**。da.gd/pp.uaにはこの制約なし。
3. **KV書き込み上限（ChatGPT指摘）**：Cloudflare KV無料は**書き込み1,000/日**。1クリック=1書き込みの現設計だと、**日1,000クリック超で計測が落ちる**。今は低トラフィックで問題ないが、**将来伸びたらD1(10万/日)へ移すべき**。これは“短さ”とは別問題だが、頭出しとして重要。
