🔁 GAS 自動反映・初期設定ガイド

この設定は「1回だけ」。以後は GASを反映.bat を押すだけでクラウドに反映されます。

これまで「コードをコピペ → 再デプロイ → トリガー設定」と手作業だったGASの反映を、PCでバッチを1回押すだけに変えます。公開URL（exec URL）は変わりませんので、アプリ側の設定変更は不要です。

コードを編集
→
GASを反映.bat
→
クラウドへ自動反映
→
トリガー再設定も自動
この初期設定は PC で行います（iPhoneではできません）。所要 5〜10分・1回だけ。
1

Apps Script API をオンにする（アカウントで1回だけ）

下のURLを開いて、「Google Apps Script API」を オン にします。

https://script.google.com/home/usersettings
※ すでにオンなら、そのままでOKです。
2

スクリプトID を確認してコピー

  1. 下のURLを開き、記録用の GAS プロジェクト（recorder）を開く
  2. 左メニューの ⚙️ プロジェクトの設定 を開く
  3. 「スクリプト ID」の値をコピー（あとで貼ります）
https://script.google.com/home
3

PC で「GAS初期設定.bat」を実行

リポジトリ直下の GAS初期設定.bat をダブルクリック。順番に次が起きます。

  1. ブラウザが開く → john.mori8k@gmail.com で承認（clasp のログイン）
  2. 黒い画面が スクリプトID を聞く → 手順2でコピーした値を貼って Enter
  3. 次に exec URL を聞く → 下をコピーして貼る（今のURL）
https://script.google.com/macros/s/AKfycbyQlrWuud5WfE3YMjoYzX2WhstTyWw_sOxVCaT8EfaFMXwi_WZzWIBKarHdtXVsY3Fj/exec
  1. ADMIN_SECRET を聞かれる → 設定していなければ何も入れず Enter（空でOK）
これで scripts/gas_deploy_config.json が自動作成され、初期設定は完了です（この設定ファイルは共有・公開されません）。
注意：途中で「clasp login に失敗」等が出たら、手順1のAPIオンを確認してからやり直してください。
4

（任意・推奨）ADMIN_SECRET を設定して門番を強くする

反映後の自動処理（トリガー再設定）を「知っている人だけ」呼べるようにする設定です。省略しても動きます。

  1. GAS プロジェクト → ⚙️ プロジェクトの設定 → スクリプト プロパティ
  2. プロパティ名に ADMIN_SECRET、値に好きな文字列を入れて保存
  3. 手順3の gas_deploy_config.json の adminSecret に同じ値を入れる（設定し直す場合は初期設定batをやり直し）

✅ 完了！ これ以降の反映のしかた

コード（コード.gs）が更新されたら、PCで GASを反映.bat を押すだけ。

  • 自動コード転送 → 既存デプロイに新バージョン反映（URLは不変）
  • 自動?ping=1 でバージョン一致を確認
  • 自動トリガー再設定＋シートのヘッダ移行

反映せずに「今クラウドが最新かどうか」だけ見たいときは、コマンドで node scripts/deploy_gas.mjs --check を実行します。

うまく反映されているかは、下のURLを開いて version が新しくなっているかで分かります。
https://script.google.com/macros/s/AKfycbyQlrWuud5WfE3YMjoYzX2WhstTyWw_sOxVCaT8EfaFMXwi_WZzWIBKarHdtXVsY3Fj/exec?ping=1
