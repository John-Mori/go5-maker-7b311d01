どのファイルを直せば、どこが変わる？

「5秒動画メーカー」の中身を、やさしく案内します。
文字や見た目を自分で直したいあなたへ。

①反映のしくみ ②全体の地図 ③文字を直す場所 ④ファイル早見表 ⑤直すときのコツ

🔄① まず大事な「反映のしくみ」

ファイルを直す前に、これだけ知っておくと安心です。

このアプリは GitHub Pages という仕組みで動いています。あなたのスマホで見えている画面は、 GitHub に置いてある ファイルそのものです。だから――

📝 ファイルを直す → 💾 GitHub に保存(push) → ⏱ 数十秒〜数分 → 📱 スマホに反映

⚠️ いちばんの注意：バージョン番号
index.html の中に ?v=104 のような番号があちこちに付いています。 これは「スマホに古い画面を表示させないためのおまじない」です。 見た目や文字を直したら、この番号を v=104 → v=105 のように 1つ増やすと、確実に新しい画面が届きます。
（クロードコードに頼むときは、この番号上げも自動でやります）

いちばん上の階層に、よく使うファイルが並んでいます。色つきが「文字・見た目に関係するファイル」です。

🖋️③ 文字・見た目を直したい人へ（ここが本題）

● 画面に出る文字のほとんどは index.html の中

タブの名前（🎬動画作成・🦋投稿…）、ボタンの文字、入力欄の上の説明、注意書き―― 画面で読める文字の大半は、この1ファイルに書かれています。

たとえば、こんな感じで書かれています👇

<!-- タブの名前 --> <button id="tabPost" class="tab">🦋 投稿</button> <button id="tabVerify" class="tab">📋 投稿履歴</button> <!-- ボタンや説明文 --> <button id="postNowBtn">🦋 今すぐ投稿</button> <div class="hint">af_id は「🔗 アフィリンク」タブの設定を流用…</div>

> と < で挟まれた部分（上の緑の文字）が、画面に出る文字です。 ここだけ書き換えれば、表示が変わります。<…> の記号や id="…" は触らないのがコツ。

● 色・大きさ・余白を変えたいなら style.css

「文字をもう少し大きく」「この色を変えたい」というのは style.css の担当です。 ファイルの先頭に色の一覧（--accent など）があり、ここを変えると全体の色味が一度に変わります。

🔎 文字が見つからないとき（中級）
一部の文字は .js ファイルの中から自動で表示されています （例：投稿本文のひな型は bluesky.js、YouTube説明欄のひな型もそこ）。 まずは index.html をエディタの 検索（Ctrl+F / ⌘+F） で直接さがし、 見つからなければ .js を探す――の順がおすすめです。
⚠️ 文字化けに注意
保存するときは必ず 文字コード「UTF-8」で。別の形式で保存すると、日本語が「縺ゅⅰ」のように化けます。 スマホで化けて見えたら、たいていこれが原因です。

🌱⑤ 自分で直すときの安全な手順とコツ

  1. 直す前にコピーを取る。うまくいかなくても元に戻せるよう、ファイルを複製してから作業すると安心。
  2. 文字だけ書き換える。<…> の記号、id="…"、class="…" は触らない。日本語の文だけを直す。
  3. UTF-8 で保存する。文字化け防止のいちばん大事なポイント。
  4. バージョン番号を1つ増やす。index.html の v=104 → v=105 に。これでスマホに新しい画面が届く。
  5. GitHub に保存(push)する。数十秒〜数分でスマホに反映。反映が遅いときは、スマホ側でページを再読み込み。
💡 迷ったら、まずここ
「あの文字どこ？」と思ったら、index.html を開いて 検索（Ctrl+F / ⌘+F） で その文字をそのまま打ち込んでみてください。たいてい一発で見つかります。
⚠️ 仕組み（.js の核）は無理に触らない
赤い核のファイルは、動きの心臓部です。文字直しの範囲を超える変更は、 クロードコードに「この文字をこう変えたい」と伝えるのがいちばん安全で早いです。
