A

STEP A：Bluesky と連携する（投稿の準備）

アカウント本体のパスワードは入れません。代わりに「アプリパスワード」という使い捨ての合言葉を1つ作って入れます。漏れても、それだけ無効にすれば本体は安全です。

アプリパスワードを発行する

Bluesky を開いて、設定からたどります。

設定（Settings）→ プライバシーとセキュリティ → アプリパスワード → アプリパスワードを追加 → 名前は何でもOK（例：go5）→ 作成
⚠️

表示される xxxx-xxxx-xxxx-xxxx はその場でコピーしてください（あとから見られません）。忘れたら作り直せばOK。

ツールを開いて「投稿する」をオンに

動画メーカーの下のほうにある 🦋 のところです。

🦋 動画作成後に Bluesky へ投稿する のチェックをオン → 下に設定欄が開きます

ハンドルとアプリパスワードを入れる

設定欄の中の ⚙ 投稿アカウント・記録設定 を開いて入力します。

ハンドル＝あなたの @ のあと（例：book-matomeex7.bsky.social）
アプリパスワード＝さっきコピーした xxxx-xxxx-xxxx-xxxx を貼り付け
✅

これで連携は完了。入力内容はあなたの端末の中だけに保存され、次回から自動で入ります。

C

STEP C：使ってみる（投稿までの流れ）

連携できたら、実際の流れはこれだけ。

写真を選ぶ

📷 写真を選ぶ / 撮る で、漫画ページなどの画像を選びます。

文字を入れる

作者名・誘導文・コメント（3段目＝動画タイトルにもなる）を入力。

プレビューを見ながら位置調整

プレビューの横の − ＋ ボタンで、文字・黒帯の位置や、黒帯の余白・段の間隔を微調整。

投稿の本文を整える

🦋 の本文ボックスには見本が入っています。自由に書き換えて、最後の行にアフィリンク（短縮しない生リンク）を貼り付けます。

アフィリンクは 🔗 アフィリンク タブで、作品URL＋af_id から作って「リンクをコピー」→ 本文の最後に貼り付け

動画を作成 → 確認 → 投稿

▶ 動画を作成 を押すと、約5秒で完成。確認ダイアログで本文を見て「OK」すると、画像つきでBlueskyに投稿されます。

B

STEP B：記録を残す（任意・あとででOK）

「投稿の短縮URLが何回開かれたか」を Google スプレッドシートに自動で貯めるしくみです。少し手順が多いので、投稿に慣れてからで大丈夫。

▶ 記録のセットアップ手順を開く

Bitly のトークンを取る（無料）

Bitly にログイン → Settings → API → トークンを発行してコピー。

⛔

このトークンは公開サイトには絶対に貼らない。次の手順で Google 側にだけ預けます。

記録用スプレッドシートを作る

Google ドライブで新規スプレッドシートを作成。URLの /d/●●●/edit の ●●●（ID） を控えます。

Apps Script にコードを貼る

script.google.com → 新しいプロジェクト → 既定のコードを消して、フォルダ内の gas/コード.gs の中身を貼り付け→保存。

カギ（プロパティ）を登録する

プロジェクトの設定（⚙）→ スクリプト プロパティに、次の名前で登録：

BITLY_TOKEN
SHEET_ID
SHARED_SECRET

値：BITLY_TOKEN＝手順1のトークン／SHEET_ID＝手順2のID／SHARED_SECRET＝好きな合言葉（任意）

ウェブアプリとして公開（デプロイ）

右上 デプロイ → 新しいデプロイ → ウェブアプリ。
「実行するユーザー：自分」「アクセス：全員」でデプロイ→初回は権限を許可→ …/exec のURLをコピー。

クリック数の自動更新をオン

関数選択で setupTrigger を選び ▶ 実行（1回だけ）。以後1時間ごとにクリック数が更新されます。

ツールに記録用URLを入れる

🦋 の ⚙ 投稿アカウント・記録設定 を開いて：

記録用URL（GAS）＝手順5の …/exec URL
共有シークレット＝手順4の SHARED_SECRET（設定した場合）
✅

これで投稿のたびに、スプレッドシートへ題名（動画タイトル）・各URL・クリック数が1行ずつ貯まります。

安全のために（だいじ）

アプリパスワードは「使い捨て」

万一漏れても、Blueskyでそのアプリパスワードだけ無効化すればOK。本体パスワードは入れません。

共有・公共の端末では入力しない

設定はその端末に保存されます。他人も使う端末では入れない／使ったら消す。

カギを人に見せない・コードに直接書かない

Bitlyトークンやアプリパスワードは、スクショやチャットに貼らない。サイトのコードにも書かない（Google側にだけ預ける）。

アフィリンクは「そのまま」

本文に貼る生リンクは加工しません＝報酬の計測(af_id)は壊れません。安心して貼ってOK。
