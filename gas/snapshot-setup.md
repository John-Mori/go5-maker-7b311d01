# YouTubeショート 再生数スナップショット — セットアップ手順

## 前提

- 既存の GAS プロジェクト（コード.gs のデプロイ先）に `snapshot.gs` を追加するだけ。
- 別デプロイ・別プロジェクトは不要。
- YouTube API キーは **GAS の Script Properties にのみ保存**する（コードやリポジトリには置かない）。

---

## 1. YouTube Data API キー（GAS 専用）を発行する

コード.gs（フロント用）とは **別のキー**を使う。

| 設定項目 | 値 |
|---|---|
| API | YouTube Data API v3 |
| アプリケーションの制限 | **なし**（GAS はリファラーを送らないため） |
| API の制限 | YouTube Data API v3 のみ |

> **理由**: GAS からのリクエストにはリファラーが付かないため、
> `john-mori.github.io` 制限付きのフロント用キーでは `403 Forbidden` になる。

---

## 2. Script Properties に YOUTUBE_API_KEY を追加

1. GAS エディタを開く
2. 左サイドの **「プロジェクトの設定」（⚙ アイコン）** をクリック
3. **「スクリプト プロパティ」** → **「プロパティを追加」**
4. プロパティ名: `YOUTUBE_API_KEY`、値: 手順1で作成した API キー → 保存

> ⚠️ API キーの値は **スクリプトのコードには絶対に書かない**。

---

## 3. GAS に snapshot.gs を追加する

1. GAS エディタのサイドバー **「ファイル」横の「＋」** → **「スクリプト」**
2. ファイル名 `snapshot`（`.gs` は自動付与される）
3. `gas/snapshot.gs` の内容をそのままペーストして保存（Ctrl+S）

---

## 4. トリガーを登録する

1. GAS エディタ上部の **「実行する関数」** ドロップダウンで `setupSnapshotTrigger` を選択
   （ファイル: snapshot）
2. **「▶ 実行」** をクリック → 権限の確認ダイアログが出たら承認
3. 実行ログに `snapshotViews トリガーを 30分ごとに登録しました。` が表示されれば成功

確認方法: 左サイドの **「時計マーク（トリガー）」** → `snapshotViews` が 30分ごとに表示されること。

---

## 5. 動作確認（手動実行）

1. 実行する関数を `snapshotViews` に切り替えて **「▶ 実行」**
2. スプレッドシートに **「再生数_管理」「再生数_スナップショット」** シートが自動作成される
3. `記録_ch1`/`記録_ch2` に YouTube URL が入っている行が `再生数_管理` に登録される
4. 再生数が `再生数_スナップショット` に記録される（ログで「N 件スナップショット完了」を確認）

---

## シート構造（自動作成）

### 再生数_管理

| 列 | 説明 |
|---|---|
| internal_id | 動画の背骨ID（記録シートの post_id） |
| youtube_id | YouTube 11文字動画ID |
| channel | `acc1` または `acc2` |
| published_at | YouTube 投稿日時（初回スナップ時に自動取得） |
| status | `active`（追跡中）/ `done`（28日超・終了） |
| first_seen | 初めてスナップした日時 |
| last_snapshot_at | 最後にスナップした日時 |
| last_view_count | 最後に記録した再生数 |

### 再生数_スナップショット

| 列 | 説明 |
|---|---|
| internal_id | 動画の背骨ID |
| youtube_id | YouTube 11文字動画ID |
| channel | `acc1` または `acc2` |
| published_at | YouTube 投稿日時 |
| snapshot_at | このスナップを取った日時 |
| elapsed_min | YouTube 投稿からの経過分数 |
| elapsed_bucket | `0-6h` / `6-24h` / `1-7d` / `7-28d` / `28d+` |
| view_count | この時点の再生数 |
| view_delta | 前回スナップからの再生数増加（初回は空白） |

---

## ティア定義（経過時間 → サンプリング間隔）

| 経過時間 | 間隔 | 理由 |
|---|---|---|
| 0〜6時間 | 30分 | 公開直後の急成長フェーズ |
| 6〜24時間 | 2時間 | 当日の伸び把握 |
| 1〜7日 | 6時間 | 週内の継続伸び |
| 7〜28日 | 24時間 | 長期テール期 |
| 28日超 | 記録終了 | status='done' に更新 |

---

## トリガー停止

追跡をやめたいとき：

```
実行する関数: deleteSnapshotTrigger → ▶ 実行
```

または GAS の「トリガー」画面から `snapshotViews` を手動削除。

---

## YouTube API クォータ見積もり

`videos.list?part=snippet,statistics` は最大50件まとめても **1ユニット消費**。

| 追跡動画数 | ユニット/日（概算） | 備考 |
|---|---|---|
| 10本 | 約 48 | 30分×48回/日、全件1バッチ |
| 50本 | 約 48 | 同上（50件以内なら1バッチ） |
| 100本 | 約 96 | 2バッチ/回 |

無料枠は 10,000ユニット/日なので、通常運用では余裕あり。

## トラブルシューティング

| 症状 | 原因・対処 |
|---|---|
| ログに `YOUTUBE_API_KEY が未設定` | Script Properties に `YOUTUBE_API_KEY` を追加する |
| YouTube API から 403 Forbidden | キーのアプリケーション制限を「なし」に設定する |
| `再生数_管理` に動画が登録されない | 記録_ch1/ch2 の「YouTube動画URL」列が空の行はスキップ（正常動作） |
| `view_delta` が空白 | 初回スナップは前回値が無いため空白（正常動作） |
| 動画が API から返ってこない | 動画が非公開・削除済みの場合スナップをスキップ（正常動作） |
