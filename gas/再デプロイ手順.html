🛠️ GAS 再デプロイ 完全ガイド

5秒動画メーカーの記録システム（Google Apps Script）

これを上から順にやればOK。所要 約5分・PC推奨

このガイドのゴール： 新しい コード.gs を貼って、本当に反映（＝再デプロイ）できたか自分の目で確認するまで。 最後の STEP 5 で ?ping=1 を開いて バージョンが出れば成功です。

🔴 まず：なぜ今までうまくいかなかったか

Apps Script の一番のワナは2つ。たぶんこれにハマっていました👇
  • ① コードを保存しても、Webアプリは古いまま。
    「💾保存」＝下書き保存なだけ。「デプロイ」して新しいバージョンを発行して初めて、実際に動くコードが入れ替わります。 → 「貼り替えて保存しただけ」だと、ずっと昔のコードが動き続けます。
  • ② 「新規デプロイ」を作るとURLが変わる。
    新しく作り直すと /exec のURLが別物になり、アプリ側の設定とズレて記録が止まります。 必ず「既存のデプロイ」を“編集（鉛筆）”して「新しいバージョン」にするのが正解。
合言葉：「貼る → 保存 → 既存デプロイを“新しいバージョン”に更新 → ping で確認」。この4つです。

📋 準備するもの

  • 貼り替える新しいコード（チャットで送った コード.gs のファイル）。開いて全文コピーできる状態に。
  • Googleアカウント john.mori8k@gmail.com でログイン済みのブラウザ。
  • PC推奨（スマホでもできますが、STEP 4 のボタンが小さく事故りやすいです）。

手順

1

エディタ（編集画面）を開く

script.google.com/home を開く

ログイン後、プロジェクト一覧から 記録用のプロジェクトをタップ／クリック。

注意：…/exec のURL（記録の送信先）は編集画面ではありません。あれは「実行用」。編集は必ず script.google.com/home から入ります。

こう見えるはず： 左側にファイル一覧、中央に コード.gs の中身（プログラム）が表示される画面。

2

コードを丸ごと貼り替えて保存

  • 中央のコード部分をクリック → 全選択（Ctrl+A / Macは ⌘+A）→ 削除。
  • 送った コード.gs の中身を全部ペースト（Ctrl+V / ⌘+V）。
  • 💾保存（Ctrl+S / ⌘+S、または上のフロッピーアイコン）。
これだけでは反映されません。 保存は下書き。必ず STEP 4 まで進めてください。
3

setupTrigger を1回だけ実行（自動更新の掃除＆再設定）

  • 上のツールバーの関数選択（プルダウン）で setupTrigger を選ぶ。
  • ▶ 実行 を押す。
  • 初回は権限の確認が出ます → アカウントを選び →「詳細」→「(安全でないページ)に移動」→「許可」。 （自分が作ったスクリプトなので問題ありません）

役割： 古い Bitly 用の自動更新を消し、新しい「Bluesky反応＋開封数」の毎時更新を入れ直します。

こう見えるはず： 下の「実行ログ」に赤いエラーが出ず、完了（白いログ）になればOK。

4

★ここが本番：デプロイ（既存を“新しいバージョン”に更新）

右上の デプロイ ▾ →

デプロイ ▾→デプロイを管理

開いた画面で、すでにある行（アクティブなデプロイ）の右にある ✏️鉛筆（編集） を押す。

デプロイを管理→既存の行の ✏️鉛筆

「バージョン」の欄を 「新しいバージョン」 に変更 → デプロイ を押す。

バージョン：新しいバージョン→デプロイ
絶対にやらないこと： 「新しいデプロイ」ボタンで作り直さない！ URLが変わってアプリと切れます。必ず “既存” を鉛筆で編集して新バージョン。
正しくできていれば： 完了画面のウェブアプリURL（…/exec）が前と同じままのはずです。変わっていたら新規を作ってしまっています（STEP 4 をやり直し）。
5

✅ 本当に反映できたか確認（これが一番大事）

下のボタンを開いてください。新しいコードが動いていれば バージョンが表示されます。

🔍 反映チェックを開く（?ping=1）
成功の表示（こうなればOK）：
{"ok":true,"version":"2026-06-25-A（Bitly撤去・upsert・開封数取込）",…,"bitly":"removed",…}
version に「2026-06-25-A」、bitly:"removed" が出ていれば反映成功です🎉
失敗のサイン：
  • version が出ない／古い文言（例 go5-maker recorder v2 だけ）→ まだ古いコードが動いています。STEP 4 をやり直し（新規デプロイにしていないか確認）。
  • 「承認が必要」等のエラーページ → STEP 3 の権限承認をやり直す。

✅ 最終チェックリスト

  • 新しい コード.gs を全文貼り替えて 💾保存した
  • setupTrigger を1回実行した（権限も承認）
  • 既存デプロイを鉛筆で編集し「新しいバージョン」でデプロイした（新規は作っていない）
  • ?ping=1 で version「2026-06-25-A」・bitly:"removed" を確認した

🎁 これで効くようになること

  • 重複行が消える：同じ動画の記録は1行にまとまる（upsert）。
  • Bitlyの毎時エラーが止まる：無料枠オーバーの発生源を撤去。短縮は da.gd / link-worker に一本化。
  • 開封数がシートに出る：YT説明欄の go5-short リンクが開かれた回数を「Bitlyクリック」列へ毎時反映。

※ BITLY_TOKEN プロパティはもう不要（残っていても無害）。

🆘 うまくいかない時

  • どのプロジェクトか分からない → 記録用スプレッドシートを開き、メニュー 拡張機能 → Apps Script から開くと確実（PCのブラウザで）。
  • 「デプロイを管理」に行が無い → まだ一度もWebアプリ公開していない可能性。デプロイ ▾ →「新しいデプロイ」→ 種類「ウェブアプリ」→ アクセス「全員」で初回作成（この時だけ新規でOK。以後は更新）。※この場合 /exec URLが新しくなるので、その値をチャットで教えてください（アプリ側に反映します）。
  • それでも version が変わらない → ブラウザのキャッシュかも。?ping=1 のURLを別タブ／プライベートタブで開き直す。

詰まったら、どのSTEPで何が表示されたかをチャットで送ってください。1タップずつ一緒に進めます。
