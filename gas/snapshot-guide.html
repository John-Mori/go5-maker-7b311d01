📊 再生数スナップショット
セットアップガイド

YouTubeショートの再生数を投稿直後から時間ごとに自動記録するシステムです。
このページを見ながら、上から順番に作業してください。

⏱ 所要時間：約10分 🔑 GAS Script Properties に設定 📋 手順5つ
1

🔑 YouTube API キーを発行する

GAS 専用・コード.gs のキーとは別に作る
⚠️
フロント用のキーと別に作る理由
GAS から YouTube API を叩くとき、リクエストにリファラーが付きません。 「john-mori.github.io 限定」の制限があるキーを使うと 403 Forbidden になります。 GAS 専用キーは「リファラー制限なし」で作ってください。
①
Google Cloud Console → 認証情報 ↗ を開き、 「認証情報を作成」→「API キー」 をクリック
②
作成されたキーの 「キーを制限」 をクリック
③
下記の通り設定する
アプリケーションの制限 → 🚫 なし
API の制限 → ✅ YouTube Data API v3 のみ
✅
「保存」してキーをコピーしておく（次の手順で使う）
2

🗝️ GAS に YOUTUBE_API_KEY を追加

Script Properties に保存する（コードには書かない）
①
GAS エディタ（コード.gs のプロジェクト）を開く
②
左サイドバーの 「⚙️ プロジェクトの設定」 をクリック
③
下にスクロール → 「スクリプト プロパティ」→「プロパティを追加」
プロパティ名： YOUTUBE_API_KEY
値　　　　　： AIza〜〜〜〜〜 （手順①でコピーしたキー）
④
「スクリプト プロパティを保存」 をクリック
🚫
API キーの値はここ（Script Properties）だけに置く。
スクリプトのコード・リポジトリ・チャットには絶対に書かない。
3

📄 snapshot.gs を GAS に追加する

リポジトリの gas/snapshot.gs をコピペするだけ
①
GAS エディタのサイドバー「ファイル」横の 「＋」→「スクリプト」 をクリック
②
ファイル名を snapshot と入力（.gs は自動で付く）して Enter
③
リポジトリの gas/snapshot.gs の内容を全選択してコピーし、 GAS の snapshot.gs にまるごと貼り付ける
④
Ctrl+S（または ⌘+S） で保存
✅
エラーが出なければ OK。ファイルタブに snapshot が増えていれば成功。
💡
コード.gs との関係：openSS_ / prop_ / headerMap_ / CH_SHEETS などの関数は同一プロジェクト内で共用されます。別ファイルに定義されていても問題ありません。
4

⏰ 30分トリガーを登録する

setupSnapshotTrigger を一度だけ実行する
①
GAS エディタ上部の 「実行する関数」ドロップダウン で
setupSnapshotTrigger
を選択（ファイルは snapshot）
②
「▶ 実行」 をクリック
※ 権限の確認ダイアログが出たら「許可」を選ぶ
③
実行ログに下記が表示されれば成功
snapshotViews トリガーを 30分ごとに登録しました。
④
左サイドの 「🕐 トリガー」 アイコンをクリックして確認
✅
snapshotViews — 時間ベース — 30分ごと
が表示されていれば登録完了
5

🧪 動作確認（手動テスト）

snapshotViews を手動で実行して確認する
①
「実行する関数」を
snapshotViews
に切り替えて 「▶ 実行」
②
スプレッドシートを開いて、以下の 2枚の新シート が自動作成されているか確認する
📋
再生数_管理 — 動画ごとの追跡状態
再生数_スナップショット — 時刻別の再生数記録
③
GAS の実行ログを確認する
// 例：YouTube URL 入りの行が3件あったとき
seedNewVideos_: 3 件を再生数_管理へ追加。
snapshotViews: 3 件スナップショット完了。
④
再生数_スナップショット シートに行が追記されていれば完成！
📝
「再生数_管理」に動画が登録されない場合
記録_ch1 / 記録_ch2 の「YouTube動画URL」列に URL が入っている行が必要です。 ウィザードで動画 URL を入力してから投稿した行に自動で入ります。
📊

自動記録のタイミング（ティア定義）

動画の経過時間で間隔が自動で変わる
YouTube 投稿からの経過時間 記録間隔 理由
0〜6時間 ⚡ 30分ごと 公開直後の急成長フェーズ
6〜24時間 🕐 2時間ごと 当日の伸びを把握
1〜7日 ⏳ 6時間ごと 週内の継続的な伸び
7〜28日 📅 24時間ごと 長期テール期
28日超 🏁 記録終了 status が done になる

トラブルシューティング

うまくいかないときはここを見る
❌
ログに「YOUTUBE_API_KEY が未設定」と出る
→ Script Properties の設定を再確認。プロパティ名のスペルを確認（大文字・アンダースコア）
❌
YouTube API から 403 Forbidden が返る
→ API キーの「アプリケーションの制限」が「なし」になっているか確認
❌
「再生数_管理」が空のまま
→ 記録_ch1 / 記録_ch2 シートの「YouTube動画URL」列に URL を入力してから再度 snapshotViews を実行
🛑
追跡を止めたいとき：deleteSnapshotTrigger を実行するか、 GAS のトリガー画面から snapshotViews を削除する
