# 投稿記録＆クリック集計（Google Apps Script）セットアップ手順

動画ツール本体（GitHub Pages）はそのまま。**この GAS を1つ用意するだけ**で、
「投稿の共有URLを Bitly 短縮 → **チャンネル別シート（記録_ch1 / 記録_ch2）に自動記録** → クリック数と
Bluesky反応（いいね/リポスト/返信）を毎時更新」が動きます。

> 記録先は **「動画記録分析テンプレート.xlsx」を取り込んだスプレッドシート**を前提にしています
> （`記録_ch1 / 記録_ch2 / 集計 / 設定` シートと名前付き範囲 `Holidays` を含む）。GASは現在のチャンネルに応じて
> `記録_chN` の該当列（post_id/投稿日時/題名/ハッシュタグ/作品cid/各URL/短縮URL/補助列）へ**列名で自動記入**します。

> クリック数の定義：**投稿の短縮URL（Bitly）が開かれた回数**。
> 本文の **アフィリンクは無改変の生リンク**なので af_id 計測は壊れません（アフィリンクのクリック実数は FANZA 管理画面が正）。

---

## 1. Bitly トークンを取得（無料）
1. https://bitly.com にログイン → **Settings → API**（または Developer settings）
2. **Generate token**（Access Token）を発行してコピー
3. ※ このトークンは GAS にだけ貼る。アプリ（公開サイト）には載せない。

## 2. 記録用スプレッドシートを用意（テンプレートを取り込む）
- **「記録分析テンプレート/動画記録分析テンプレート.xlsx」**を Google ドライブにアップロード → 右クリック →
  **「Googleスプレッドシートで開く」**（数式・名前付き範囲ごと変換されます）。
- そのスプレッドシートの URL `/d/●●●/edit` の **●●●部分（スプレッドシートID）** を控える。
- シート `記録_ch1 / 記録_ch2 / 集計 / 設定` が含まれていることを確認（GASはこの `記録_chN` に書き込みます）。
  もし無い場合でも、GASが最低限のヘッダーで自動作成しますが、集計・day-type判定のためテンプレ取り込みを推奨。

## 3. Apps Script プロジェクトを作成
1. https://script.google.com → **新しいプロジェクト**
2. 既定の `Code.gs` を全消去し、本フォルダの **`コード.gs` の中身を貼り付け**て保存
3. 左メニュー **プロジェクトの設定（⚙）→ スクリプト プロパティ** で以下を追加：

| プロパティ名 | 値 | 必須 |
|---|---|---|
| `BITLY_TOKEN` | 手順1のトークン | ✅（短縮URLを使うなら） |
| `SHEET_ID` | 手順2のスプレッドシートID | ✅ |
| `SHARED_SECRET` | **設定しない（空のまま）** | ✕ |

> ⚠️ **`SHARED_SECRET` は設定しないでください。** 現在のアプリは記録時にシークレットを送らないため、
> ここに値を入れると「bad_secret」で記録が弾かれます（無人予約も同様）。

## 4. ウェブアプリとしてデプロイ
1. 右上 **デプロイ → 新しいデプロイ**
2. 種類：**ウェブアプリ**
3. 次のユーザーとして実行：**自分**
4. アクセスできるユーザー：**全員**（※ URLを知っている人だけが叩ける。SHARED_SECRET でさらに保護）
5. **デプロイ** → 初回は権限承認（自分のGoogleアカウントを許可）
6. 表示される **ウェブアプリ URL（…/exec）** をコピー

## 5. クリック数・反応の自動更新を有効化
- エディタの関数選択で **`setupTrigger`** を選び、▶ 実行（1度だけ）
  → 以後 **1時間ごと**に `refreshClicks`（Bitlyクリック）と `refreshEngagement`（Bluesky いいね/リポスト/返信）が走り、
  各 `記録_chN` の該当列を更新します。
- すぐ反映したい時は **`refreshClicks`** / **`refreshEngagement`** を手動実行。
- 反応は Bluesky 公開API（`public.api.bsky.app`・未認証）で取得。クリックは直近250件のみ更新（実行時間/レート対策）。

## 6. アプリ側に設定を入力
動画ツールの「🦋 投稿」タブ →「⚙ 投稿アカウント・記録設定」を開き：
- **記録用URL（Google Apps Script）** … 手順4の `…/exec` URL（**入れるのはこれだけ**）

> ここに入れるのは **GASウェブアプリの `…/exec` URL** です。**スプレッドシートのURLではありません。**
> スプレッドシートは手順3で `SHEET_ID` としてGAS側に登録済みなので、アプリには貼りません。
> 現在のアプリに「共有シークレット」欄はありません。

これで「動画を作成 → 確認 → Bluesky投稿 → スプレッドシートに1行記録」まで自動です。

## 記録シート（記録_chN）でGASが自動記入する列
- **post_id / 投稿日時 / 題名(コメント) / ハッシュタグ / 作品cid / Bluesky投稿URL / 短縮URL**（投稿時）
- **いいね / リポスト / 返信 / Bitlyクリック**（毎時更新）
- 補助列 **Bitly_ID / post_uri / クリック更新日時 / 反応更新日時**（システム用・手で触らない）
- ※ **インプレッション/CTR/視聴回数/維持率（YouTube）** と **FANZA成約（発生/確定/報酬）** は手入力。
  `曜日 / day-type / 時間帯スロット / 承認率 / クリック率 / CVR / EPC / RPM` はテンプレの数式で自動計算。

## ☁️ Phase5：無人で予約投稿（タブを閉じても投稿）

「⏰予約投稿」で**☁️無人で予約する**をONにすると、予約をGASへ送り、**時間トリガーが指定時刻に自動投稿**します（端末は閉じてOK）。

### 追加セットアップ
1. **アプリ側**：Bluesky 設定 →「アプリパスワード」を発行（既出）。
2. **GASのスクリプトプロパティに、投稿用アカウントを追加**：

| プロパティ名 | 値 | 必須 |
|---|---|---|
| `BSKY_HANDLE` | 例：`yourname.bsky.social` | ✅（無人予約に） |
| `BSKY_APP_PW` | アプリパスワード（xxxx-xxxx-xxxx-xxxx） | ✅（無人予約に） |

3. エディタで **`setupReservationTrigger`** を選び ▶ 実行（1回だけ）。以後 **5分ごと**に `runReservations` が走り、期限到来の予約を投稿します。すぐ試すなら `runReservations` を手動実行。
4. 初回実行時に **Google ドライブの権限**を承認（画像の一時保存に使用。投稿後は自動でゴミ箱へ）。

### しくみ・注意
- 予約は **「予約」シート**に貯まり、`pending → posting → posted/error` と進みます。画像は base64 で受け取り **ドライブに一時保存**→投稿時に取得→投稿後にゴミ箱へ。
- 投稿アカウントは **GAS側の `BSKY_HANDLE/BSKY_APP_PW`**（端末のアプリ設定ではなく）。秘匿情報はGASにのみ置く方針。
- 失敗は「予約」シートの**エラー列**に記録。`SHARED_SECRET` は前述のとおり**未設定のまま**にする（設定すると無人予約も弾かれます）。
- 「無人OFF」の予約は従来どおり**タブを開いている間だけ**動作（端末のアカウント設定を使用）。

## トラブル時
- 記録されない：`…/exec` URL が正しいか／デプロイのアクセス権が「全員」か／`SHARED_SECRET` の綴り一致を確認
- クリック数が0のまま：`refreshClicks` を手動実行して権限を承認。Bitly反映に時間差あり
- デプロイを更新したら**新しいURL**になることがある（その場合はアプリ側のURLも更新）
