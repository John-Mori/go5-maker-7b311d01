# 投稿記録＆クリック集計（Google Apps Script）セットアップ手順

動画ツール本体（GitHub Pages）はそのまま。**この GAS を1つ用意するだけ**で、
「投稿の共有URLを Bitly 短縮 → Googleスプレッドシートに自動記録 → クリック数を毎時更新」が動きます。

> クリック数の定義：**投稿の短縮URL（Bitly）が開かれた回数**。
> 本文の **アフィリンクは無改変の生リンク**なので af_id 計測は壊れません（アフィリンクのクリック実数は FANZA 管理画面が正）。

---

## 1. Bitly トークンを取得（無料）
1. https://bitly.com にログイン → **Settings → API**（または Developer settings）
2. **Generate token**（Access Token）を発行してコピー
3. ※ このトークンは GAS にだけ貼る。アプリ（公開サイト）には載せない。

## 2. 記録用スプレッドシートを用意
- Google ドライブで新規スプレッドシートを作成（名前は任意）
- URL の `/d/●●●/edit` の **●●●部分（スプレッドシートID）** を控える
- （シート1枚目はそのままでOK。スクリプトが「記録」シートを自動作成します）

## 3. Apps Script プロジェクトを作成
1. https://script.google.com → **新しいプロジェクト**
2. 既定の `Code.gs` を全消去し、本フォルダの **`コード.gs` の中身を貼り付け**て保存
3. 左メニュー **プロジェクトの設定（⚙）→ スクリプト プロパティ** で以下を追加：

| プロパティ名 | 値 | 必須 |
|---|---|---|
| `BITLY_TOKEN` | 手順1のトークン | ✅ |
| `SHEET_ID` | 手順2のスプレッドシートID | ✅（推奨） |
| `SHARED_SECRET` | 任意の合言葉（アプリ側と同じ文字列） | 任意 |

## 4. ウェブアプリとしてデプロイ
1. 右上 **デプロイ → 新しいデプロイ**
2. 種類：**ウェブアプリ**
3. 次のユーザーとして実行：**自分**
4. アクセスできるユーザー：**全員**（※ URLを知っている人だけが叩ける。SHARED_SECRET でさらに保護）
5. **デプロイ** → 初回は権限承認（自分のGoogleアカウントを許可）
6. 表示される **ウェブアプリ URL（…/exec）** をコピー

## 5. クリック数の自動更新を有効化
- エディタの関数選択で **`setupTrigger`** を選び、▶ 実行（1度だけ）
  → 以後 **1時間ごと**に `refreshClicks` が走り、スプレッドシートのクリック数列を更新します
- すぐ反映したい時は **`refreshClicks`** を手動実行

## 6. アプリ側に設定を入力
動画ツールの「🦋 Bluesky自動投稿」カードで：
- **記録用URL（Google Apps Script）** … 手順4の `…/exec` URL
- **共有シークレット** … 手順3の `SHARED_SECRET`（設定した場合のみ）

これで「動画を作成 → 確認 → Bluesky投稿 → スプレッドシートに1行記録」まで自動です。

---

## スプレッドシートの列
| 記録日時 | 題名(動画タイトル) | アフィリンク | 投稿URL | 短縮URL(投稿) | Bitly_ID | クリック数 | クリック更新日時 |
|---|---|---|---|---|---|---|---|

- **題名(動画タイトル)** … 3段目コメント（=ファイル名にもなる文字）
- **クリック数** … 投稿短縮URL（Bitly）の開封数。`refreshClicks` が毎時更新

## トラブル時
- 記録されない：`…/exec` URL が正しいか／デプロイのアクセス権が「全員」か／`SHARED_SECRET` の綴り一致を確認
- クリック数が0のまま：`refreshClicks` を手動実行して権限を承認。Bitly反映に時間差あり
- デプロイを更新したら**新しいURL**になることがある（その場合はアプリ側のURLも更新）
