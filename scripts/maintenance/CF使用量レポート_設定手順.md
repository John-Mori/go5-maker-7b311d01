# Cloudflare 使用量レポート — 設定手順(Chami用・5分)

毎朝、Worker別のリクエスト数がDiscord(改修α)に1行で届くようになる。
**8万/日を超えたら警告**が付く。2026-07-16の「無料枠超過をメールで知った」を二度と起こさないため。

設定するまでは**何も起きない**(スクリプトは静かに終わる)ので、急がなくてよい。

---

## 1. APIトークンを作る(閲覧のみ・最小権限)

1. https://dash.cloudflare.com/profile/api-tokens を開く
2. **「トークンを作成」→ 一番下の「カスタムトークン」→「作成を開始」**
3. 名前: `go5-usage-read`(何でもよい)
4. **アクセス許可**をこう設定する(これだけ):

| 項目 | 選ぶもの |
|---|---|
| 1つ目 | **アカウント** → **Account Analytics** → **読み取り** |

   - ★これ以上の権限を付けない。**読み取り専用なので、漏れてもWorkerを壊されたり課金されたりしない**。
5. **アカウントリソース**: 「Johnl.Jmori8k@gmail.com's Account」を選ぶ
6. 「概要に進む」→「トークンを作成」
7. 表示されたトークンを**コピー**(この画面でしか見られない)

## 2. トークンを置く(★チャットには貼らない)

**Discordにも、Claudeのチャットにも貼らないこと。** 自分でファイルに保存する。

1. メモ帳を開く → コピーしたトークンを貼る → 次の場所に保存:
   `D:\SougouStartFolder\go5-maker\local\cf_api_token.txt`
2. もう1つメモ帳を開く → **アカウントID**を貼る → 次の場所に保存:
   `D:\SougouStartFolder\go5-maker\local\cf_account_id.txt`
   - アカウントIDの調べ方: Cloudflareのダッシュボードを開いた時のURL
     `https://dash.cloudflare.com/<ここの32桁>/...` の部分。
     または「Workers & Pages」の右側「アカウントの詳細」→ アカウントID。

※ `local/` はgitignore済み=**GitHubには絶対に上がらない**。

## 3. 動作確認

黒い画面(コマンドプロンプト)で:

```
cd /d D:\SougouStartFolder\go5-maker
python scripts\maintenance\cf_usage_report.py --dry
```

- `☁️ Cloudflare 2026-07-16: go5-sync 3.2k / r2 4.1k … 計 7.7k / 枠 100.0k` のように出れば成功。
  (`--dry` は表示するだけでDiscordへ送らない)
- `未設定のためスキップ` → ファイルの場所か名前が違う。
- `トークンの権限(Account Analytics:Read)を確認して` → 手順1-4を見直す。

うまくいったら、あとは**毎日自動で届く**(日次メンテの常駐に組み込み済み)。

