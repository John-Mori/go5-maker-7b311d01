自動投稿と記録の仕組み

「動画を作成 → Bluesky に投稿 → クリック数をスプレッドシートに記録」。
いま何がどこで動いているかを、専門用語なしで図にしました。

🎬 ボタンひとつで「動画作成 → 写真つきで Bluesky に投稿 → 記録」まで自動。

投稿には あなたが選んだ元写真 と 無改変のアフィリンク（＝報酬がちゃんと計上される状態）が入ります。 投稿のあと、その投稿への短縮URLを Bitly で作り、Googleスプレッドシートに 「動画タイトル・各URL・クリック数」を1行ずつためていきます。

① 登場人物（どこで何が動く？）

あなたのスマホ／PCのブラウザを中心に、外の3つのサービスとやり取りします。動画づくり自体は 端末の中 で完結します。

実線＝あなたの端末から送る／破線＝Google側からBitlyへ（カギを端末に出さないため）

② ボタンを押してから記録されるまで

③ 「クリック数」は何を数えている？（重要）

数えているのは 「投稿の短縮URL（Bitly）が開かれた回数」 です。 ＝ 別の場所に貼ったその短縮リンクから、何人が投稿にアクセスしたか、の目安。

⚠️

正直なところ：投稿に貼った「生のアフィリンク」が実際に何回押されたかは、こちらからは取得できません （Blueskyにその仕組みがないため）。アフィリンクの本当のクリック数は FANZA の管理画面が正です。 スプレッドシートの数字は「どれだけ投稿に人が来たか」の目安、と捉えてください。 アフィリンクには一切手を加えないので、報酬計測（af_id）は壊れません。

今回作った Bluesky自動投稿・記録機能。いまはMacBookの中にだけあります。 公開URLに出すには git push が必要です。

あなたの設定待ち

記録の保管庫（GAS）と Bitly。手順どおりに1回セットアップ＋アプリに2つのURLを入れたら稼働します。

🗺️

登場するURLは3つ：①ツール本体（公開サイト）／②localhost:8000（手元の確認用だけ）／ ③Googleの記録用URL（あなたがデプロイ後に発行）。混同しないように。

⑤ あなたが用意するもの（1回だけ）

  • 1Bluesky の「アプリパスワード」
    設定 → プライバシーとセキュリティ → アプリパスワード で発行。通常のパスワードは使いません。
  • 2Bitly の無料トークン
    Bitly → Settings → API で発行。これは公開サイトには載せず、Google側にだけ預けます。
  • 3記録用のGoogleスプレッドシート
    新規作成してIDを控えるだけ。表の見出しは自動で作られます。
  • 4Google Apps Script をデプロイ
    用意した gas/コード.gs を貼って「ウェブアプリ」として公開 → …/exec のURLを取得。
  • 5アプリに入力
    投稿設定カードに「記録用URL」と「共有シークレット」を入れて完了。

⑥ セキュリティの考えどころ

「公開サイト＝中身が誰でも見える」前提で設計しています。秘密の鍵をサイトのコードに書かないことが肝心です。

✅ 鍵は端末とGoogle側だけ

アプリパスワード等はあなたの端末（localStorage）に、BitlyトークンはGoogle側に保管。公開サイトのコードには一切書きません。

✅ アプリパスワードは使い捨て可能

万一漏れても、Bluesky側でそのアプリパスワードだけ失効すればOK。アカウント本体のパスワードは入力しない設計です。

✅ アフィリンクは無改変

リンクを書き換えたり間に挟んだりしないので、規約・報酬計測の面でも安全側です。

⚠️ 共有端末・公共PCは注意

設定は端末に保存されます。他人も使う端末では入力しない／使用後に「リセット」を。

⚠️ 記録用URLは「合言葉」で守る

GASのURLは知られると誰でも書き込めます。必ず 共有シークレット（SHARED_SECRET）を設定して、他人の書き込みを弾いてください。

⛔ やってはいけないこと

トークンやアプリパスワードを HTMLやJSに直書きしてpushしない。スクショやチャットに貼らない。GitHubに秘密ファイルを上げない。

1. ちゃんとした投稿プレビュー画面

かんたん

いまは確認が簡易ダイアログ。本文・添付画像・リンクを大きく確認してから送れる画面にすると安心感が上がります。

2. アプリ内に投稿履歴を表示

かんたん

スプレッドシートを開かなくても、直近の投稿・短縮URL・クリック数をアプリ内の一覧で見られるように。

3. 添付画像を「元写真／合成プレビュー」で選べる

かんたん

今回は元写真に固定。トグル1つで切り替えられると用途に応じて使い分けできます。

4. ハッシュタグ・定型文の管理

ふつう

固定文を複数テンプレ化して使い分け。曜日やジャンルで本文を出し分けるなど。

5. 失敗時のリトライと分かりやすいエラー表示

ふつう

ネットワーク不調・画像が重い等を切り分けて表示し、ワンタップで再送できるように。

6. セキュリティ強化（合言葉の必須化・鍵の定期更新）

ふつう

共有シークレット未設定だと記録できないように強制。アプリパスワード／トークンの定期更新の促しも。

7. 予約投稿・複数作品まとめ投稿

大きめ

時間指定の自動投稿や、複数URLを順番に投稿。GAS側のしくみを拡張する規模になります。

8. X（旧Twitter）対応

大きめ

Xへの自動投稿は別途サーバー＋OAuthが必須（このツール単体では不可）。やるなら専用の小さなサーバーを足す構成になります。
