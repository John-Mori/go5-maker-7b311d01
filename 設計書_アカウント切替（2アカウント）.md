# 設計書：アカウント切替（2アカウント対応）

> 対象：5秒動画メーカー スマホ版（現行 v=21）。本機能で v=22 に上げる。
> 目的：**2つのアカウントを切り替えて、動画作成（背景）／投稿の対象を選べる**ようにする。

---

## 1. 目的・背景

- 運用アカウントを2つに増やす。アカウントごとに**背景動画**を変える。
- ユーザーが画面上で「**いまどちらのアカウントで作業するか**」を選べる方式にする（動画作成・投稿の両方に効く土台）。
- アカウント2の Bluesky / YouTube は**まだ未作成**のため、**当面はアカウント1のものを流用**（＝保留）。
  本設計はアカウントの**枠組み**を用意し、将来それぞれの資格情報を差し込める形にしておく。

---

## 2. アカウント定義（唯一の真実＝app.js の `ACCOUNTS`）

```js
const ACCOUNTS = {
  acc1: {
    label: "アカウント1",
    bg: "assets/bg_main.mp4",       // 既存の背景
    bluesky: null,                  // null = 共有設定（既存の localStorage キー）を使う
    youtube: null,                  // 未実装（将来）
  },
  acc2: {
    label: "アカウント2",
    bg: "assets/bg_account2.mp4",   // 今回追加した背景（22857_1280x720→取り込み済み）
    bluesky: null,                  // 保留：当面アカウント1（＝共有設定）を流用
    youtube: null,                  // 保留：当面アカウント1を流用
  },
};
```

- **アカウント間で異なるのは現状「背景動画」だけ**。それ以外（テキスト3段・位置調整・アフィID・Bluesky/GAS設定）は
  両アカウント共通＝アカウント1と同条件で作成・投稿する。
- `bluesky/youtube` が `null` の間は**共有設定を使う**（＝今までどおり）。将来 `acc2.bluesky = {handle, appPw}` を入れたら、
  そのアカウントだけ個別資格情報で投稿する拡張に進める（本設計の範囲外＝保留）。

---

## 3. 現在アカウントの保持

- `localStorage["current_account"]`：`"acc1"` | `"acc2"`（既定 `"acc1"`）。
- 起動時に復元。切替時に保存。**端末ごと**に保持（他のスマホ/PCには影響しない）。

---

## 4. UI

ヘッダー直下・タブバーの**すぐ上**に、常時見える「アカウント切替バー」を置く（全タブ共通で見える）。

```
[ アカウント: (●アカウント1) ( アカウント2 ) ]   ← セグメント型の2ボタン
```
- 選択中のボタンを強調（`.active`）。タップで切替。
- 既存デザイントークン（`--card --accent --line` 等）に準拠。タップ領域は十分に確保。
- 補助文（任意）：「背景動画がアカウントごとに変わります。投稿アカウントは現在アカウント1を共用（2つ目は準備中）」を小さく表示。

---

## 5. 動作仕様

1. **切替時**（acc1⇄acc2）：
   - `current_account` を保存。
   - 背景 `<video id="bg">` の `src` を選択アカウントの `bg` に差し替え → `bg.load()` → `bg.play()` → プレビュー再描画。
   - セレクタの `.active` を更新。
   - 拡張用に `document` へ `account-changed`（`detail:{id}`）を dispatch（将来 bluesky.js 等が購読可能。今は購読者なし）。
2. **起動時**：保存値（無ければ `acc1`）に従って背景 `src` を設定してから初期描画。
   - 既存 index.html の `<video id="bg" src="assets/bg_main.mp4">` を初期値とし、保存が `acc2` の場合のみ差し替える（無駄な再読込を避ける）。
3. **動画作成（make）/プレビュー**：現在アカウントの背景がそのまま使われる（背景参照は `<video id="bg">` 一本のままなので、既存の合成・録画ロジックは無変更で動く）。
4. **投稿（Bluesky）**：当面アカウント1の共有設定で投稿（acc2 の `bluesky` が `null` のため）。挙動は現行どおり＝**bluesky.js は無変更**。

> 背景動画 `bg_account2.mp4` は 1280×720（横長）。合成は既存の cover クロップ（中央切り出し）で 9:16 に収める（`drawFrame` は無変更で対応）。

---

## 6. 影響範囲（最小・既存機能を壊さない）

| ファイル | 変更 |
|---|---|
| `index.html` | アカウント切替バーを追加（タブバー上）。全アセットの `?v=21`→`?v=22`。 |
| `app.js` | `ACCOUNTS` 定義・現在アカウントの復元/保存・背景 `src` 切替・`account-changed` 発火。**描画/座標ロジックは無変更**。 |
| `style.css` | 切替バーのスタイル（`.acct-bar` 等）。`?v=22`。 |
| `assets/bg_account2.mp4` | 追加済み（アカウント2の背景）。 |
| `bluesky.js` / `scheduler.js` / `integration.js` / `verify*` / `gas/` | **無変更**（投稿・スケジュール・検証は現行のまま）。 |

---

## 7. 受け入れ条件

- [ ] 画面上部でアカウント1/アカウント2を切り替えられ、選択が一目で分かる。
- [ ] アカウント2を選ぶと**背景動画が `bg_account2.mp4` に変わり**、プレビュー・書き出しに反映される。
- [ ] アカウント1に戻すと従来の `bg_main.mp4` に戻る。
- [ ] 選択は再読み込みしても保持される（`current_account`）。
- [ ] 背景以外（テキスト3段・位置調整9コントロール・アフィID・Bluesky/GAS・スケジュール）は**両アカウントで共通かつ従来どおり**動く（回帰ゼロ）。
- [ ] Bluesky 投稿は当面アカウント1の設定で動く（acc2 も同設定を流用）。
- [ ] 既存の座標系規約（1080×1920 比率ベース）・`?v=` キャッシュ運用を踏襲。
